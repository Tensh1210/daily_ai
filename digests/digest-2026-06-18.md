## 📄 Research Papers — Top Picks
- **VISUALSKILL: Multimodal Skills for Computer-Use Agents** — arXiv cs.CL · https://arxiv.org/abs/2606.18448
  Computer-use agents still fail on long-horizon tasks and unseen software; existing skill libraries store skills as text only, ignoring the visual nature of GUI work. VISUALSKILL is a hierarchical multimodal skill organized as a central index over per-topic files, consumed via a `load_topic` MCP tool that fetches relevant text and figures on demand, built from a two-stage pipeline of authored docs plus live UI exploration. A Claude Code CLI agent on Claude Opus 4.6 hits 0.456 average on CUA-World and OSExpert-Eval, a +15-point gain. **Why it matters:** Shows visually grounded, MCP-served skills materially lift real computer-use agents on hard software tasks.
- **Towards Scalable Customization and Deployment of Multi-Agent Systems for Enterprise Applications** — arXiv cs.CL · https://arxiv.org/abs/2606.18502
  Production multi-agent systems are blocked by domain-customization needs plus high latency and inference cost. The paper proposes a two-stage framework: Agentic Model Customization (continual pretraining + SFT + preference optimization) to adapt a compact model while keeping agentic skill, then Inference Optimization combining speculative decoding and FP8 quantization with targeted calibration for cheap serving. **Why it matters:** A concrete recipe for turning costly agentic prototypes into deployable, low-latency enterprise systems.
- **Decoupling Search from Reasoning: A Vendor-Agnostic Grounding Architecture for LLM Agents** — arXiv cs.CL · https://arxiv.org/abs/2606.18947
  Native search grounding bundles retrieval policy, provider, evidence injection, cost, and latency behind one model-provider boundary, hard to inspect or port and prone to "Search-Induced Verbosity." Decoupled Search Grounding (DSG) moves grounding outside the reasoning model via an MCP-compatible gateway exposing provider routing, source-aware rendering, fallback, retrieval-depth control, and exact-plus-semantic caching as first-class controls. Across five frontier models on SimpleQA/FreshQA/HotpotQA, DSG exposes a stronger frontier than native search except on recency-sensitive FreshQA. **Why it matters:** Turns search grounding into a tunable, portable infrastructure layer instead of an opaque vendor feature.
- **Beyond the Current Observation: Evaluating Multimodal LLMs in Controllable Non-Markov Games** — HF Papers · https://huggingface.co/papers/2606.19338
  Deploying multimodal models as closed-loop policies increasingly requires acting on observations no longer visible, but existing benchmarks expose full state or test recall only post-episode. RNG-Bench isolates a model's ability to reconstruct past observations mid-interaction via two games — Matching Pairs (recall briefly revealed cards) and 3D Maze (integrate egocentric views into a map) — under controlled grid-size, pattern, and modality axes. **Why it matters:** Cleanly measures the hidden-state memory that agentic multimodal models need but rarely have tested in isolation.
- **TopBench: A Benchmark for Implicit Predictive Reasoning in Tabular Question Answering** — arXiv cs.CL · https://arxiv.org/abs/2604.28076
  Most table QA is solved by extraction or simple aggregation, but many real queries are implicitly predictive, requiring inference of unobserved answers from historical patterns. TopBench provides 779 samples across four sub-tasks (single-point prediction, decision making, treatment-effect analysis, complex filtering), demanding outputs that span reasoning text and structured tables. **Why it matters:** Exposes a predictive-reasoning gap in LLMs that standard table-QA benchmarks completely miss.
- **CoreMem: Riemannian Retrieval and Fisher-Guided Distillation for Long-Term Memory in Dialogue Agents** — arXiv cs.CL · https://arxiv.org/abs/2606.18406
  Personalized dialogue agents need long-term memory but choke on consumer hardware (e.g., 8 GB VRAM), and isotropic cosine retrieval plus heuristic compression lack theory and suffer hubness and syntactic fragmentation. CoreMem is an edge-cloud architecture unified by information geometry: Riemannian retrieval swaps cosine for a locally adaptive Fisher-Rao metric to penalize hub memories, paired with Fisher-guided distillation for compression. **Why it matters:** Brings a principled, resource-efficient memory stack to on-device personal agents.
- **MCompassRAG: Topic Metadata as a Semantic Compass for Paragraph-Level Retrieval** — arXiv cs.CL · https://arxiv.org/abs/2606.18508
  RAG faces a chunking trade-off: fine chunks improve precision but blow up search space and cost, while large chunks mix topics and weaken dense similarity. MCompassRAG enriches chunk representations with topic-level metadata as a semantic compass, selecting evidence by topic signals rather than only noisy cosine similarity, targeting fast and precise retrieval over large heterogeneous corpora. **Why it matters:** A lightweight metadata layer that improves deep-research retrieval without exploding latency or index size.
- **PragReST: Self-Reinforcing Counterfactual Reasoning for Pragmatic Language Understanding** — arXiv cs.CL · https://arxiv.org/abs/2606.18624
  LLMs excel at math and logic but still default to literal readings on pragmatic inference. PragReST is a self-supervised framework that builds pragmatic QA data, generates counterfactual reasoning traces, and trains models via SFT plus RL — with no human labels and no distillation from a stronger teacher. It improves over backbones and task-specific baselines across four pragmatic benchmarks (PragMega, Ludwig, MetoQA, AltPrag). **Why it matters:** Teaches implied-meaning reasoning self-sufficiently, a capability that resists scaling alone.
- **Beyond Reward Engineering: A Data Recipe for Long-Context Reinforcement Learning** — arXiv cs.CL · https://arxiv.org/abs/2606.18831
  Long-context reasoning is essential for autonomous agents reasoning over long trajectories, yet RL work fixates on reward engineering while training data stays scarce. The authors show a simple data recipe alone, with a minimal outcome-based GRPO setup, substantially improves long-context reasoning by targeting retrieval, multi-evidence synthesis, and reasoning across ~14K curated examples. Gains hold across Qwen3-4B/8B/30B-A3B. **Why it matters:** Reframes long-context RL as a data problem, offering a cheap, reproducible path to better agentic reasoning.
- **GraphPO: Graph-based Policy Optimization for Reasoning Models** — arXiv cs.CL · https://arxiv.org/abs/2606.18954
  RLVR samples responses independently and optimizes from final answers, causing redundant exploration of similar intermediate steps and sparse-reward credit assignment; tree methods help via shared prefixes but still expand branches independently. GraphPO models reasoning as a graph so branches reaching similar states share information instead of repeating exploration, yielding finer-grained signals. **Why it matters:** Cuts wasted compute and sharpens step-level credit in RL for reasoning models.
- **LLMZero: Discovering Adaptive Training Strategies for RL Post-Training via LLM Agents** — arXiv cs.CL · https://arxiv.org/abs/2606.18388
  RL post-training strategies are dataset-dependent: capacity parameters grow monotonically across stages while regularization oscillates with training dynamics, so fixed schedules can't track non-stationary exploration-exploitation trade-offs. LLMZero uses LLM agents to tree-search over training trajectories, diagnosing pathologies at each checkpoint and proposing coordinated multi-parameter transitions, improving over base by 9–140% across four GRPO tasks. **Why it matters:** Automates the brittle hand-tuning of RL post-training schedules via agentic search.
- **GateMem: Benchmarking Memory Governance in Multi-Principal Shared-Memory Agents** — arXiv cs.CL · https://arxiv.org/abs/2606.18829
  Memory benchmarks assume single users, ignoring shared assistants in hospitals, workplaces, and households where many principals write and query a common memory under different roles and scopes. GateMem jointly evaluates utility on long-horizon requests, access control across authorization boundaries, and active forgetting after deletion, spanning medical, office, education, and household domains with leak-target annotations. **Why it matters:** Surfaces governance, access-control, and forgetting failures that single-user memory benchmarks never test.
- **ToolGrad: Efficient Tool-use Dataset Generation with Textual "Gradients"** — arXiv cs.CL · https://arxiv.org/abs/2508.04086
  Prior tool-use datasets generate a query first, then annotate complex tool calls (e.g., DFS), causing frequent annotation failures and low efficiency. ToolGrad inverts this: it builds valid tool-use chains iteratively via textual "gradients," then synthesizes matching user queries — an answer-first approach producing ToolGrad-500 with more complex tool use, lower cost, and near-100% pass rate. Models trained on it beat expensive baselines and proprietary LLMs. **Why it matters:** A cheaper, higher-yield way to manufacture tool-use training data for agents.
- **ActMem: Bridging the Gap Between Memory Retrieval and Reasoning in LLM Agents** — arXiv cs.CL · https://arxiv.org/abs/2603.00026
  Current memory frameworks treat agents as passive recorders that retrieve without understanding implications, failing where reasoning is needed. ActMem turns unstructured dialogue history into a structured causal-and-semantic graph, then uses counterfactual reasoning and commonsense completion to deduce implicit constraints and resolve conflicts between past states and current intent, evaluated on the new ActMemEval. **Why it matters:** Makes agent memory actionable rather than just retrievable, improving long-term decision quality.
- **Notation Matters: A Benchmark Study of Token-Optimized Formats in Agentic AI Systems** — arXiv cs.CL · https://arxiv.org/abs/2605.29676
  Agentic systems exchange tool schemas, results, and invocations as structured data, and the default JSON imposes heavy token overhead. The paper benchmarks token-optimized alternatives TOON and TRON inside end-to-end agentic loops across four benchmarks (BFCL, MCPToolBenchPP, MCP-Universe, StableToolBench) and five open-weight LLMs, decoupling input compression from generation effects. **Why it matters:** Tests whether compact tool-call formats actually save tokens without hurting agent accuracy in realistic loops, not just isolated tasks.

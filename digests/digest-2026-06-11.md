## 📄 Research Papers — Top Picks
- **InternVideo3: Agentify Foundation Models with Multimodal Contextual Reasoning** — Ziang Yan et al. (Shanghai AI Lab) · https://huggingface.co/papers/2606.12195
  InternVideo3 targets long-horizon multimodal tasks that open-source agentic work has largely ignored in favor of text-dominant settings. Its Multimodal Contextual Reasoning (MCR) treats video understanding as a closed-loop process over a shared, evolving context of observations, instructions, reasoning, tool actions, and memory — framing long-video understanding as evidence accumulation and verification. **Why it matters:** Closed-loop, tool-using video agents extend the agentic paradigm beyond text into sustained temporal reasoning.
- **When More Documents Hurt RAG: Mitigating Vector Search Dilution with Domain-Scoped, Model-Agnostic Retrieval** — Nabaraj Subedi et al. · https://arxiv.org/abs/2606.11350
  Names a production failure mode — "vector search dilution" — where scaling a deployed corpus from 54 to 1,128 documents (88,907 chunks) dropped accuracy from 75% to below 40%, even with hybrid dense+sparse retrieval. Proposes MASDR-RAG, a multi-agent scoped domain retrieval scheme, evaluated on 200 expert-validated queries across five LLM backbones. **Why it matters:** Hard production numbers showing RAG degrades with corpus growth, plus a model-agnostic mitigation.
- **Agents All the Way Down: A Methodology for Building Custom AI Agents from Substrate to Production** — Marc Alier Forment et al. · https://arxiv.org/abs/2606.11869
  Observes that the practice of building custom AI agents end-to-end — agents living inside their own application, with their own data, tools, security boundaries, and audit trail — exists only scattered across podcasts, blogs, and leaked system prompts. Writes it down as a formal methodology covering function-calling APIs, MCP, and paired code agents from substrate to production. **Why it matters:** First published end-to-end methodology for the custom-agent engineering practice most teams improvise today.
- **Organize then Retrieve: Hierarchical Memory Navigation for Efficient Agents (HORMA)** — Hao-Lun Hsu et al. · https://arxiv.org/abs/2606.11680
  HORMA tackles agent statelessness on long-horizon tasks: instead of lossy compression or similarity-based retrieval, it organizes experience into a file-system-like hierarchical structure and navigates it at retrieval time. This preserves the temporal structure and causal dependencies that multi-step agentic tasks need. **Why it matters:** File-system-style hierarchical memory is a practical alternative to embedding retrieval for long-running agents.
- **SkillJuror: Measuring How Agent Skill Organization Changes Runtime Behavior** — Zhiyu Chen et al. · https://arxiv.org/abs/2606.11543
  Separates what an agent Skill says from how it is organized, comparing Progressive Disclosure (a concise root file pointing to resources on demand) against a normalized flat baseline. In an 82-task SkillsBench study with semantically controlled variants and matched multi-trial evaluations, Progressive Disclosure changes runtime behavior before aggregate outcomes shift. **Why it matters:** Empirical evidence that skill file organization — not just content — changes agent behavior, directly relevant to anyone authoring skills.
- **MODF-SIR: A Multi-agent Omni-modal Distilled Framework for Social Intelligence Reasoning** — Shang Ma et al. · https://arxiv.org/abs/2606.12018
  A multi-agent collaborative framework on a lightweight multimodal LLM for social intelligence reasoning, with knowledge distillation augmenting both training and inference. Long-tail social events are localized, extracted, and rendered as formatted explicit text so they are not drowned out by head events and environmental noise during tokenization. **Why it matters:** Distilled multi-agent pipelines let small multimodal models punch above their weight on socially grounded reasoning.
- **Vision-Language-Action Jump-Starting for Reinforcement Learning Robotic Agents (VLAJS)** — Angelo Moroncelli et al. · https://arxiv.org/abs/2604.13733
  VLAJS bridges sparse Vision-Language-Action guidance with on-policy RL: the VLA provides generalist task-level reasoning to jump-start exploration, while RL delivers the high-frequency closed-loop control VLAs cannot. This attacks the exploration and credit-assignment problems of long-horizon manipulation with sparse rewards. **Why it matters:** A clean recipe for combining pretrained VLA priors with RL where each compensates for the other's weakness.
- **World Model Self-Distillation: Training World Models to Solve General Tasks** — Sebastian Stapf et al. · https://huggingface.co/papers/2606.12072
  Pretrained video generators show emergent task-solving ability but depend on detailed text descriptions for planning. This framework combines self-distillation with RL: a VLM generates a candidate task and step-by-step solution from an unlabeled scene image, which conditions a pretrained video diffusion model — eliminating costly paired task-execution videos. **Why it matters:** Scalable self-supervision could turn video generators into general task-solving world models without expensive labeled data.
- **Agentic Environment Engineering for Large Language Models: A Survey** — Jiachun Li et al. · https://huggingface.co/papers/2606.12191
  Systematically surveys agentic environments through an engineering lifecycle lens — modeling, synthesis, evaluation, and application — covering representative environments across eight attributes and eight domains. Fills the gap of systematic categorization in a space crucial to driving continual agent capability evolution. **Why it matters:** Environments are the training ground and bottleneck for agent progress; this is the map of that territory.
- **NightFeats @ MMU-RAGent NeurIPS 2025: A Context-Optimized Multi-Agent RAG System** — Quentin Fever et al. · https://arxiv.org/abs/2606.11199
  Award-winning (Best Dynamic Evaluation) multi-agent RAG system that decomposes knowledge synthesis into retrieval, curation, and composition phases, each governed by explicit intermediate representations and handoff contracts. Introduces temporal-semantic reranking, bounded contradiction reconciliation, and citation-preserving composition as architectural primitives. **Why it matters:** Contract-driven phase decomposition is a transferable blueprint for production multi-agent RAG.
- **Agent Skill Evaluation and Evolution: Frameworks and Benchmarks** — Kexin Ding et al. · https://arxiv.org/abs/2606.11435
  Surveys the shift from isolated skill creation to automated, evaluation-driven skill evolution as agent skill libraries scale. Categorizes evolution into four paradigms — execution feedback, trajectory distillation, compression, and reinforcement learning — and maps the evaluation landscape for utility, quality, and safety. **Why it matters:** A framework for systematically evolving skill libraries instead of hand-curating them.
- **ISE: An Execution-Grounded Recipe for Multi-Turn OS-Agent Trajectories** — Siyuan Luo et al. · https://arxiv.org/abs/2606.11520
  ISE (Intent → Simulate → Execute) synthesizes OS-agent training data in three stages: ~44K deduplicated structured intents from a Persona×Domain×Task×Complexity framework, multi-turn user-agent interaction via a role-locked user simulator, and grounded tool execution. Jointly captures structured intents, multi-turn delegation, and grounded execution — properties absent from existing datasets. **Why it matters:** A reproducible recipe for the multi-turn, execution-grounded data OS agents actually need.
- **"Do Not Mention This to the User": Detecting and Understanding Malicious Agent Skills in the Wild** — Yi Liu et al. · https://arxiv.org/abs/2602.06547
  Systematic security analysis of 98,380 agent skills from two major community registries, using static pattern matching plus dynamic behavioral verification. Identifies 157 skills with confirmed malicious behavior spanning 632 distinct vulnerabilities across 13 attack techniques — skills that execute with full user privileges. **Why it matters:** First large-scale ground truth that the agent-skill supply chain is already being exploited.
- **CRANE: Constrained Reasoning Injection for Code Agents via Nullspace Editing** — Mingzhi Zhu et al. · https://arxiv.org/abs/2605.14084
  Instruct checkpoints are tool-disciplined but weaker planners; Thinking checkpoints plan well but over-deliberate and degrade agent performance. CRANE is a training-free parameter-editing method that treats the Thinking–Instruct delta as a pool of candidate reasoning edits, injecting planning ability into the Instruct backbone via nullspace editing without breaking tool discipline. **Why it matters:** Training-free surgery to get Thinking-level planning with Instruct-level tool reliability in code agents.
- **Runtime Skill Audit: Targeted Runtime Probing for Agent Skill Security** — Tu Lan, Chaowei Xiao · https://arxiv.org/abs/2606.11671
  Agent skills can look benign in documentation and code yet turn harmful only under particular requests, local assets, persistent state, or multi-step tool interactions — making static vetting brittle. RSA audits what the skill-mediated agent actually does under targeted runtime conditions, profiling risk-relevant interfaces instead of running generic tests. **Why it matters:** Dynamic, behavior-based skill auditing is the missing complement to static registry scans.

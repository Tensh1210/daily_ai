## 📄 Research Papers — Top Picks
- **RF-Agent: A Practical Framework for Building Language Agents for RFIC Design** — Xing, He, Wang, Ni et al. · https://arxiv.org/abs/2607.18772
  RF-Agent attacks the data scarcity blocking LLMs in radio-frequency circuit design by using a multi-agent Question-Thinking-Solution-Answer pipeline to distill seven canonical RF textbooks into an 11,000-sample reasoning dataset plus a multiple-choice benchmark. The authors then compare supervised fine-tuning against three RAG configurations (semantic, keyword, hybrid) across several model families. Domain SFT gives the largest gains, especially for small and mid-size models, while semantic retrieval is the best RAG variant. **Why it matters:** A reusable recipe for bootstrapping a domain agent from textbooks when no dataset exists — and evidence that SFT still beats RAG for deep domain reasoning.

- **AI Tool Discovery at Scale: All You Need is DNS** — Enhao Chen, Yulin Shao · https://arxiv.org/abs/2607.18242
  ToolDNS proposes retrofitting semantic tool discovery onto DNS instead of building yet another centralized registry, encoding functional intent and organizational trust into a hierarchical namespace so O(N) semantic search becomes O(log N) name resolution. Three protocol-compliant extensions carry the design: partially unfolded names, EDNS0 intent payloads, and logical subdomains. On a new 33,688-tool benchmark spanning MCP, A2A, REST, and Skill protocols, it cuts per-query search space by 95.26% while matching state-of-the-art retrieval accuracy, with UDP-native latency orders of magnitude below HTTP registries. **Why it matters:** If agent tool ecosystems reach millions of tools, discovery needs internet-scale infrastructure — and DNS already is that infrastructure.

- **Frontier AI performance across the business disciplines: a case-grounded benchmark of knowledge work and analytical reasoning** — Patel, Hosanagar, Krishnan, Callison-Burch, Lakhani · https://arxiv.org/abs/2607.16057
  The authors argue existing benchmarks test recall, narrow QA, math, and coding/tool-use but not the analytical knowledge work professionals actually do: synthesizing messy information, judging under uncertainty, adversarial multi-stakeholder thinking, and producing defensible structured analysis. They build BusinessCaseBench from hundreds of business-school case questions across eighteen disciplines, each graded against a rubric derived from the instructor's own case solution. Frontier models already score highly against those rubrics, with large within-family gains over two years. **Why it matters:** Shifts the "can AI do white-collar work" question from anecdote to rubric-graded measurement, and the answer is already uncomfortable.

- **SkillCorpus: Consolidating and Evaluating the Open Skill Ecosystem for Real-World LLM Agents** — Wang, Yao, Sun, Hu et al. · https://arxiv.org/abs/2607.15557
  SkillCorpus crawls ~821,000 public agent SKILL files and filters them through a multi-stage pipeline down to 96,401 skills organized by a 16-class taxonomy and three quality facets (utility, robustness, safety), paired with a fine-tuned retrieval-and-selection stack. Evaluation spans three benchmarks (SkillsBench, GDPVal, QwenClawBench), two harnesses, and two open backbones. Integrating the corpus improves all three benchmarks, up to +7.5 pp on SkillsBench, with gains traced to a coverage boundary and a harness boundary. **Why it matters:** First end-to-end evidence of when a curated community skill library actually helps an agent — and where the ceiling is.

- **AgentTrails: Towards Trust and Reuse for Agentic Tasks** — Wu, Castelo, Liu, Silva, Freire · https://arxiv.org/abs/2607.18816
  AgentTrails converts raw agent trajectories from chronological logs into structured provenance graphs, modeling tool calls as computational actions and their inputs/outputs as data artifacts. Multiple runs can be placed on a shared canvas and joined into a quotient graph that aligns recurring tools, artifacts, and dependencies across executions. On real trajectories this surfaces hidden dataflow dependencies and recurring tool-use patterns, supporting pattern extraction and skill abstraction. **Why it matters:** Treating agent runs as dataflow rather than logs is what makes comparing, debugging, and reusing agent work tractable.

- **AgentDebugX: An Open-Source Toolkit for Failure Observability, Attribution, and Recovery in LLM Agents** — Zhu, Ye, Han, Zhao et al. · https://huggingface.co/papers/2607.18754
  AgentDebugX frames agent debugging as a closed loop — Detect, Attribute, Recover, Rerun — with a DeepDebug core doing multi-turn root-cause diagnosis via global trajectory understanding, structure-guided investigation, and cross-examination. On the Who-and-When benchmark it hits 28.8% exact agent-and-step attribution on qwen3.5-9b versus 21.7% for the strongest single-pass baseline; on GAIA it repairs 13 of 73 failed tasks in one rerun (vs 4–6 for decoupled self-correction), lifting accuracy from 55.8% to 63.6%. It ships as a Python library, CLI, web console, and installable agent skill, plus an opt-in Error Hub for sharing scrubbed failure bundles as debugging memory. **Why it matters:** The step where an agent error surfaces is rarely the step that caused it — attribution tooling is what turns agent failures into fixable bugs.

- **NexForge: Scaling Agent Capabilities through Requirement-Driven Task Synthesis for LLMs** — Zhao, Lei, Xi, Zheng et al. · https://huggingface.co/papers/2607.14186
  NexForge generates agent training data from high-level capability requirements instead of from predefined tools or repos, avoiding the substrate bias of existing pipelines. It investigates real-world demand to build scenarios and task profiles, compiles distribution-aware directives, auto-constructs the needed files/dependencies/runtime configs, then synthesizes expert rollouts into SFT trajectories. Without domain-specific infrastructure it produces 3.6K terminal and 2K office tasks, lifting Qwen3.5-35B-A3B Base from 22.5% to 52.0% on Terminal-Bench 2.0 and 813→1338 Elo on GDPval; at 43.2K terminal tasks it reaches 58.4%. **Why it matters:** Agent capability increasingly scales with synthetic task supply, and this removes the manual substrate engineering that gated it.

- **Artificial Intelligence for Mathematical Reasoning: An Integrated Survey of Language Models, Neuro-symbolic Systems, and Verified Discovery** — Raiyan, Kabir, Mahmud, Hasan, Ananiadou · https://arxiv.org/abs/2606.08728
  A unified survey tracing math reasoning from rule-based word-problem solvers and template geometry systems through neural generation and LLM prompting to today's reasoning models, multi-agent systems, neuro-symbolic provers, and verified discovery workflows. It organizes the field along four axes: informal reasoning over text/diagrams, formal reasoning in proof assistants (autoformalization, tactic prediction, compiler-guided repair, proof search), mathematical discovery, and the inference/training techniques (CoT, tool use, process reward models, RLVR) linking generation to verification. It also catalogs benchmarks and examines saturation and contamination. **Why it matters:** The single best current map of where verification-coupled reasoning actually works versus where benchmarks are already saturated.

- **SkillSight: Seeing Through Shared Descriptions for Accurate Skill Retrieval** — Xiao, Ji, Li, Liu et al. · https://arxiv.org/abs/2607.18785
  SkillSight observes that skill descriptions are highly templated — shared boilerplate recurs across skills, inflating dense similarity scores and drowning the discriminative signal. It proposes a training-free calibration in two spaces: Semantic Background Calibration estimates a background subspace from generic IDF-identified tokens, and Lexical Evidence Calibration downweights shared background tokens. On SRA-Bench and SkillBench-Supp it improves Recall@10 by up to 20.21 pp over the original dense retriever and wins overall end-to-end. **Why it matters:** As skill libraries grow, retrieval quality — not skill quality — becomes the bottleneck, and this fixes it with no training.

- **One Student, Many Teachers: Multi-Task On-Policy Distillation via Soft-Prompt Privileged Context** — Ma, Zhu, Jiang, Xiao · https://arxiv.org/abs/2607.18293
  Existing on-policy self-distillation teachers either inject privileged context at the input (causing post-hoc rationalization) or fine-tune weights (accumulating drift and forgetting). This method's teacher differs from the student only by a learnable soft prompt trained on gold pairs with the backbone frozen, preserving the student's exact representational geometry. Multi-task training routes each example to its own soft-prompt teacher so one student absorbs K teachers in parallel, with all prompts discarded at inference. On Qwen3-1.7B-Base and Phi-4-mini across Science, Tool Use, Biology, and Math it matches or exceeds full fine-tuning with orders of magnitude fewer trained parameters, and unlike sequential SFT it preserves general capability. **Why it matters:** A cheap way to teach one small model several skills at once without the catastrophic forgetting that sequential SFT causes.

- **Cross-Agent Campaign Attribution: Linking Asynchronous Attacks Across LLM Agents** — Park, Choi, Kim, Kang · https://arxiv.org/abs/2607.18826
  Agent defenses are usually evaluated one session at a time, but real attacks spread across independent agents, teams, and runtimes so each local guardrail sees only a fragment. The paper formalizes cross-agent asynchronous campaign attribution and introduces Asynchronous Attribution Fingerprint Vectors, a proxy-side protocol scoring pairwise campaign similarity from observable tool-use, timing, and prompt residue. On the new SCD-v1 benchmark it reaches 0.82 pairwise AUC while per-session detectors and chunked LLM judges stay near chance; structural and stylometric residue carry the strongest signal. **Why it matters:** Multi-agent deployments need campaign-level detection, because per-session guardrails provably cannot see a distributed attack.

- **An LLM-powered Agentic Recommendation System for Connected TV Content Discovery** — Shi, Wang, Tran, Xu et al. · https://arxiv.org/abs/2607.09988
  Traditional and unified-generative recommenders consume structured behavioral signals with fixed schemas and cannot naturally absorb unstructured context like trending topics, breaking news, or cultural events without bespoke feature engineering. This system uses LLM reasoning to synthesize heterogeneous signals directly, but — acknowledging LLMs still trail classical ML on retrieval efficiency, personalization precision, and scalability — wraps them in an agentic architecture that orchestrates specialized components per sub-task. It targets Connected TV content discovery in production. **Why it matters:** A realistic hybrid pattern: LLMs for messy context reasoning, classical components for the parts they still lose on.

- **FormGym: Doing Paperwork with Agents** — Toles, Singh, Song, Yu · https://arxiv.org/abs/2506.14079
  FormGym benchmarks pure-image form filling — no OCR, no typeset PDF text, no DOM — across 432 fields in 55 documents and 3 tasks, requiring 236 user features. Baseline vision-language-action models score under 1% in most conditions, mainly from poor localization, while GUI agents manage only 10.6–68.0% at high cost and latency. The authors contribute FieldFinder, a tool that helps LLMs decide where on the form to place text, raising performance in all six study conditions (up to 2% → 56%). **Why it matters:** Localization, not comprehension, is the wall for document agents — and a small targeted tool moves it more than a bigger model does.

- **Node-as-Agent: Graph Agentic Network** — Guo, Zhu, Jiao, Liu et al. · https://arxiv.org/abs/2508.00429
  ReaGAN replaces GNNs' fixed aggregation with node-level autonomy: each node acts as an agent that plans its next action from its own memory, enabling adaptive message propagation instead of uniform neighbor averaging. This addresses both the imbalance in node informativeness and the blindness of local message passing to globally relevant but distant nodes, with RAG letting nodes retrieve semantically related content across the graph. It reaches competitive few-shot in-context performance on a frozen LLM backbone with no fine-tuning. **Why it matters:** Shows agentic planning can substitute for learned aggregation in graph learning, without training a graph model at all.

- **LinguistAgent: A Reflective Multi-Model Platform for Automated Linguistic Annotation** — Bingru Li · https://arxiv.org/abs/2602.05493
  LinguistAgent packages LLM annotation for humanities and social-science research behind a usable platform, pairing an Annotator with an optional Reviewer to simulate peer review. It supports comparative experiments across prompt engineering (zero-shot/few-shot/CoT), RAG, and fine-tuning, with real-time token-level F1 and Cohen's kappa against human gold standards. The authors validate it by replicating a published metaphor-identification study; code is released. **Why it matters:** Annotation is the real bottleneck in social-science research, and reviewer-in-the-loop agents with live agreement metrics make LLM annotation auditable rather than assumed.

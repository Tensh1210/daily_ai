## 📄 Research Papers — Top Picks
- **Dive into Claude Code: The Design Space of Today's and Future AI Agent Systems** — arXiv cs.CL · https://arxiv.org/abs/2604.14228
  The paper reverse-engineers Claude Code's publicly available architecture and compares it against two independent open-source agent systems, OpenClaw and Hermes Agent, that tackle similar design problems. It maps out how these systems structure tool execution, shell access, and file editing loops, then draws out a broader design space for agentic coding tools. The analysis highlights recurring architectural choices (permission gating, context management, tool-call loops) that show up across independently-built agents. **Why it matters:** A rare independent architectural teardown of Claude Code gives concrete design patterns worth studying for anyone building agentic coding tools.
- **SkillDAG: Self-Evolving Typed Skill Graphs for LLM Skill Selection at Scale** — arXiv cs.AI · https://arxiv.org/abs/2606.03056
  As agents accumulate large skill libraries, picking the right subset stops being a similarity-matching problem — skills depend on, conflict with, specialize, or duplicate each other in ways invisible to embedding search or full enumeration. SkillDAG models these inter-skill relationships as a typed directed graph so selection becomes structural rather than purely semantic. The graph self-evolves as new skills are added, keeping selection tractable at scale. **Why it matters:** Directly relevant to any system (like Claude Skills) managing growing, interdependent skill libraries where naive similarity search breaks down.
- **OmniGAIA: Towards Native Omni-Modal AI Agents** — arXiv cs.CL · https://arxiv.org/abs/2602.22897
  Current multimodal LLM agents are mostly confined to bi-modal (vision-language) interaction, missing the unified perception across vision, audio, and language that human intelligence uses alongside reasoning and tool use. OmniGAIA proposes a native omni-modal agent architecture aiming to unify perception across modalities with reasoning and tool usage in one cognitive loop rather than bolting modalities on separately. **Why it matters:** Points toward agents that can natively reason across audio/vision/text instead of stitching together separate single-modality pipelines.
- **AgenticSTS: A Bounded-Memory Testbed for Long-Horizon LLM Agents** — HF Papers · https://huggingface.co/papers/2607.02255
  Long-horizon agent memory is usually just appending every observation, tool call, and reflection to the prompt, which is simple but turns context into a jumbled mixture where no single memory component's effect can be isolated. AgenticSTS introduces a bounded-memory contract and testbed to instrument and measure how different memory-retention strategies affect long-horizon agent behavior in a controlled way. **Why it matters:** Gives a concrete testbed for evaluating memory-management strategies instead of relying on ad hoc "just append everything" context growth.
- **PACE: A Proxy for Agentic Capability Evaluation** — HF Papers · https://huggingface.co/papers/2607.02032
  Full agentic evaluations on benchmarks like SWE-Bench or GAIA can cost thousands of dollars and take days, unlike fast, cheap non-agentic benchmarks that test isolated capabilities like reasoning or code generation. PACE investigates whether performance on cheap non-agentic benchmarks can serve as a reliable proxy signal for expensive agentic benchmark outcomes. **Why it matters:** If validated, this could let teams cheaply estimate agentic performance before committing to full, costly agent-benchmark runs.
- **AgenticDataBench: A Comprehensive Benchmark for Data Agents** — HF Papers · https://huggingface.co/papers/2607.01647
  LLM-based data agents are emerging as a way to automate the labor-intensive work of turning heterogeneous raw data into actionable insights, but there's no comprehensive way to evaluate them. AgenticDataBench introduces a broad benchmark suite specifically targeting data-science agent tasks, covering the pipeline from raw data ingestion to insight generation. **Why it matters:** Fills a gap in standardized evaluation for the fast-growing category of autonomous data-analysis agents.
- **ChemGraph-XANES: An Agentic Framework for XANES Simulation and Curation** — arXiv cs.AI · https://arxiv.org/abs/2604.16205
  Computational XANES (X-ray absorption near-edge structure) analysis, used to interpret coordination environments and oxidation states, is often bottlenecked by workflow complexity rather than the underlying simulation method itself. ChemGraph-XANES wraps the simulation and curation pipeline in an agentic framework that automates the workflow orchestration, reducing the manual overhead of running routine computational XANES at scale. **Why it matters:** A concrete example of agentic tooling being applied to accelerate a specific, previously workflow-bottlenecked scientific computing task.
- **ContraFix: Skill-Enhanced Contrastive Runtime Analysis for Vulnerability Repair** — arXiv cs.CL · https://arxiv.org/abs/2605.17450
  Automated vulnerability repair is hard because tools usually only get failure artifacts, not repair guidance — traditional analysis gives suspicious locations or constraints but is costly to configure per bug. ContraFix combines a reusable skill library with contrastive runtime analysis, comparing failing and passing executions to extract actionable repair signals for LLM-driven patching. **Why it matters:** Moves automated vulnerability repair from static hint-following toward runtime-evidence-driven patching, which should generalize better across bug classes.
- **COMFYCLAW: Self-Evolving Skill Harnesses for Image Generation Workflows** — arXiv cs.AI · https://arxiv.org/abs/2607.01709
  As image-generation workflows become repeated and domain-specific, agents need to recall workflow patterns, execution constraints, and user preferences across runs rather than rebuilding them each time. COMFYCLAW introduces self-evolving skill harnesses that let an agent accumulate and reuse ComfyUI-style workflow skills, improving consistency and reducing repeated setup effort. **Why it matters:** A practical pattern for making creative-tool agents remember and reuse successful workflow configurations instead of re-deriving them each session.
- **MMIR-TCM: Memory-Integrated Multimodal Inference and Retrieval for TCM Clinical Decision Support** — arXiv cs.AI · https://arxiv.org/abs/2607.01814
  Traditional Chinese Medicine diagnosis via tongue inspection suffers from subjectivity and poor reproducibility, and multimodal AI struggles here due to the semantic gap between visual tongue features and diagnostic text. MMIR-TCM proposes a memory-integrated multimodal retrieval-and-inference system to bridge that gap for syndrome differentiation and prescription generation. **Why it matters:** Shows multimodal retrieval-augmented inference extending into a clinical domain with historically poor ground-truth structure.
- **ContextNest: Verifiable Context Governance for Autonomous AI Agent** — arXiv cs.AI · https://arxiv.org/abs/2607.02116
  Autonomous agents increasingly rely on external knowledge stores, but most retrieval pipelines give relevance scores without durable guarantees of provenance, version identity, integrity, or point-in-time reconstruction. ContextNest formalizes this gap as "context governance" and offers an open specification plus reference implementation for verifiable, auditable retrieval. **Why it matters:** Addresses a real production gap: agents making decisions off retrieved context that can't be audited or reproduced after the fact.
- **ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning** — arXiv cs.AI · https://arxiv.org/abs/2607.02509
  Even as context windows grow, LLMs often fail to actually use relevant evidence already present in the input — a gap between context access and effective context utilization. ReContext introduces a harness that recursively replays and re-surfaces evidence during reasoning, forcing the model to revisit and integrate long-context information rather than skimming past it. **Why it matters:** Targets the well-known "lost in the middle" failure mode directly with a reasoning-time fix rather than just a bigger context window.
- **A Practice Auditing Framework for Large Language Model Use: Collective Empiricism, Pseudo-Rational Cognition, and Governance of AI-Generated Content** — arXiv cs.CL · https://arxiv.org/abs/2607.01248
  As LLMs are used for knowledge work, code, writing, and agent-based automation, users often get structured answers, plans, and judgments without sufficient domain practice to evaluate them. This paper proposes a practice-auditing framework to govern LLM use and AI-generated content, framing the issue as "pseudo-rational cognition" that needs collective empirical checks. **Why it matters:** A governance-angle paper worth tracking as organizations formalize policies for trusting AI-generated judgments in professional workflows.
- **Ophiuchus: Incentivizing Tool-augmented "Think with Images" for Joint Medical Segmentation, Understanding and Reasoning** — arXiv cs.AI · https://arxiv.org/abs/2512.14157
  Medical multimodal LLMs generate step-by-step textual reasoning but struggle with clinical tasks needing dynamic, iterative focus on fine-grained visual regions. Ophiuchus equips a medical MLLM with tool-augmented "think with images" capability, letting it actively zoom/segment while reasoning rather than reasoning over a static image description. **Why it matters:** Extends tool-use agent patterns into medical vision-language reasoning, where iterative visual inspection is often clinically necessary.
- **CaP-X: A Framework for Benchmarking and Improving Coding Agents for Robot Manipulation** — arXiv cs.AI · https://arxiv.org/abs/2603.22435
  "Code-as-Policy" is a promising complement to data-intensive Vision-Language-Action methods for robot control, but how well code-generating agents actually perform as autonomous manipulation controllers is underexplored. CaP-X provides an open-access framework for systematically benchmarking and improving Code-as-Policy agents on robot manipulation tasks. **Why it matters:** Standardizes evaluation for a promising alternative to end-to-end VLA robot policies: agents that write executable control code instead.

## 📄 Research Papers — Top Picks
- **Keep Policy Gradient in Charge: Sibling-Guided Credit Distillation for Long-Horizon Tool-Use Agents** — Tian et al. · https://arxiv.org/abs/2606.12634
  Long-horizon tool-use RL spreads a single trajectory-level reward across many reasoning, API, and answer tokens, while naive token-level self-distillation rehearses teacher behavior without knowing which actions the verifier actually rewards — amplifying useful skills and harmful shortcuts alike. The authors show this can silently destroy tool use and propose sibling-guided credit distillation to assign denser, verifier-aware credit. **Why it matters:** Credit assignment is the core bottleneck for training reliable long-horizon tool agents; this pinpoints a failure mode of a popular shortcut.

- **MiniMax Sparse Attention** — Xunhao et al. (MiniMax) · https://huggingface.co/papers/2606.13392
  Introduces MSA, a blockwise sparse attention built on Grouped Query Attention where a lightweight Index Branch scores key-value blocks and selects a Top-k subset per GQA group, enabling group-specific sparse retrieval over ultra-long contexts. Targets agentic workflows, repository-scale code reasoning, and persistent memory that need attention over hundreds of thousands to millions of tokens. **Why it matters:** Practical sparse attention from a frontier lab directly attacks the quadratic cost blocking million-token agent deployments.

- **PersonaDrive: Human-Style Retrieval-Augmented VLA Agents for Closed-Loop Driving Simulation** — Mahmoud et al. · https://arxiv.org/abs/2606.12616
  Closed-loop driving simulators usually fill traffic with near-identical rule-based or single-mode learned agents. PersonaDrive instead conditions vision-language-action traffic agents on retrieval from demonstrations where humans were explicitly asked to drive in a given style, rather than post-hoc style labels or LLM-inferred reward weights. **Why it matters:** Style-grounded RAG for embodied agents makes simulation traffic more realistic — a transferable recipe for persona-conditioned agents beyond driving.

- **Topical Phase Transitions in Artificial Intelligence Research** — Rasul et al. · https://arxiv.org/abs/2606.12828
  Analyzing 80,814 main-track papers from ACL, CVPR, ICLR, ICML, and NeurIPS (2017–2025), the authors show major AI topics advance through abrupt phase transitions: marginal for years, then surging across venues within one to three years, as LLMs and diffusion models did. They derive an early-warning signature for emerging topics. **Why it matters:** A quantitative early-warning signal for "the next LLM-sized wave" is useful for research bets and tooling roadmaps.

- **Agents-K1: Towards Agent-native Knowledge Orchestration** — Zong et al. · https://arxiv.org/abs/2606.13669
  Argues research agents have advanced agent orchestration while neglecting knowledge orchestration: papers get reduced to abstracts and flat citation edges, losing entities, claims, evidence, and method lineages. Agents-K1 is an end-to-end pipeline converting raw documents into agent-native scientific knowledge graphs. **Why it matters:** Structured, claim-level knowledge graphs are a plausible upgrade over chunk-based RAG for serious research agents.

- **G-Long: Graph-Enhanced Memory Management for Efficient Long-Term Dialogue Agents** — Minjun et al. · https://arxiv.org/abs/2606.13115
  Long-term dialogue consistency suffers from lossy unstructured memory or expensive LLM-based processing. G-Long uses a fine-tuned small language model to maintain a graph-structured memory, cutting latency while preserving information across long conversations. **Why it matters:** Offloading memory management to a small specialized model is a practical cost/latency win for production dialogue agents.

- **SkillCAT: Contrastive Assessment and Topology-Aware Skill Self-Evolution for LLM Agents** — Kunfeng et al. · https://arxiv.org/abs/2606.13317
  Current skill self-evolution pipelines learn from one trajectory per task, merge skill patches unchecked, and load the whole skill corpus at inference. SkillCAT is training-free: it samples multiple trajectories per task and contrasts same-task success/failure pairs to extract causal evidence for what makes a skill work, organized topology-aware. **Why it matters:** Directly relevant to agent skill libraries (à la Claude skills) — contrastive validation before merging skills is a sane fix for skill-corpus rot.

- **HyperTool: Beyond Step-Wise Tool Calls for Tool-Augmented Agents** — Yaxin et al. · https://arxiv.org/abs/2606.13663
  Identifies an execution-granularity mismatch: deterministic tool workflows get unfolded into many model-visible atomic calls, burning context and forcing the model to manage low-level dataflow. HyperTool is a unified MCP-style interface that changes the model-visible unit of execution to whole workflows. **Why it matters:** Batching deterministic sub-workflows out of the reasoning trace is a concrete path to cheaper, more reliable MCP agents.

- **Pythagoras-Prover: Advancing Efficient Formal Proving via Augmented Lean Formalisation** — Josh et al. · https://arxiv.org/abs/2606.12594
  An open-source family of compute-efficient Lean theorem provers spanning autoregressive models at 4B and 32B parameters plus a first proof-of-concept diffusion-based 4B prover, built for practical budgets where verified proof data is scarce and proof-search traces are long. **Why it matters:** Open, affordable formal-proving models broaden access to verified reasoning beyond big-compute labs.

- **TerraBench: Can Agents Reason Over Heterogeneous Earth-System Data?** — Dat et al. · https://arxiv.org/abs/2606.13148
  Weather/climate foundation models forecast well but don't reason in language; LLMs reason but can't operate on gridded physical data, satellite imagery, and simulator outputs. TerraBench benchmarks grounded Earth-system reasoning across these heterogeneous inputs. **Why it matters:** A serious testbed for tool-using scientific agents on high-dimensional real-world data, far beyond text-only benchmarks.

- **Perceive, Interact, Reason: Building Tool-Augmented Visual Agents for Spatial Reasoning** — Chan et al. · https://arxiv.org/abs/2606.12830
  VLMs remain weak at spatial reasoning that requires active evidence acquisition. PERIA is a tool-augmented visual agent that interleaves perception tools, interaction, and reasoning across map reasoning, visual probing, and vision reconstruction tasks instead of relying on implicit encoder representations. **Why it matters:** Active, tool-driven visual evidence gathering is the emerging recipe for closing VLMs' spatial-reasoning gap.

- **EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments** — Junda et al. · https://huggingface.co/papers/2606.13681
  Most agent benchmarks assume static environments; EvoArena models environment change as sequences of progressive updates across terminal, software, and social domains. The companion EvoMem is a patch-based memory paradigm recording memory evolution so agents stay aligned with shifting task conditions. **Why it matters:** Real deployments drift — benchmarking and patching agent memory against change addresses a gap static evals hide.

- **Reward Modeling for Multi-Agent Orchestration** — King et al. · https://arxiv.org/abs/2606.13598
  Training orchestrators for LLM multi-agent systems is hard due to scarce supervision and cost. OrchRM is self-supervised: it mines intermediate artifacts from multi-agent executions to build win-lose pairs for Bradley-Terry reward-model training, evaluating orchestration quality without human annotations. **Why it matters:** Annotation-free reward signals for orchestration quality could make multi-agent coordinators trainable at scale.

- **Evoflux: Inference-Time Evolution of Executable Tool Workflows for Compact Agents** — Kush et al. · https://arxiv.org/abs/2606.12674
  Small LMs as tool agents must discover tools from live MCP catalogs, satisfy schemas, and preserve dependencies — and often emit plausible workflow graphs that fail at execution. Evoflux evolves executable tool workflows at inference time instead of relying on small-corpus distillation. **Why it matters:** Making compact models reliable on MCP-style tool use is the cost lever for deploying agents at scale.

- **AgentBeats: Agentifying Agent Assessment for Openness, Standardization, and Reproducibility** — Xiao et al. · https://arxiv.org/abs/2606.13608
  Agent evaluation is fragmented across fixed LLM-centric harnesses that create test-production mismatch. AgentBeats advocates Agentified Agent Assessment: judge agents evaluate participants through standardized protocols — A2A for task management and MCP for tool access. **Why it matters:** Standardizing agent evals on A2A + MCP would make cross-framework agent comparisons reproducible for the first time.

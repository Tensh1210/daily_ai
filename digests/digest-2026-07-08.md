## 📄 Research Papers - Top Picks
- **CanvasAgent: Enabling Complex Image Creation and Editing via Visual Tool Orchestration** - arXiv cs.AI · https://arxiv.org/abs/2607.05465
  Complex image work - generate, localize, segment, edit, composite, read text, enhance - rarely fits one model. CanvasAgent orchestrates a toolbox of specialist vision models so a multimodal agent actively transforms visual state across steps rather than just perceiving it, shifting from perception-augmented reasoning to manipulation-centered creation. **Why it matters:** A concrete blueprint for tool-orchestrating agents that edit images through multi-step visual workflows.
- **ArtisanCAD: An Industrial-Level CAD Agent with Expert-Grounded Knowledge Distillation** - arXiv cs.AI · https://arxiv.org/abs/2607.05750
  Industrial CAD needs long-horizon procedural modeling, robust feature dependencies, and production-grade B-Rep geometry that text-to-CAD methods miss on ambiguous prompts. ArtisanCAD distills expert procedural knowledge (e.g. CATIA-style operation sequences) into a CAD agent that turns high-level design intent into editable parametric geometry. **Why it matters:** Shows domain-expert distillation turning LLM agents into usable engineering-grade design tools.
- **From Passive Retrieval to Active Memory Navigation: Learning to Use Memory as a Structured Action Space** - arXiv cs.AI · https://arxiv.org/abs/2607.05794
  Most agent memory systems expose history through passive retrieval, leaving the model a consumer of pre-selected evidence. NapMem instead organizes user history into a linked multi-granularity 'memory pyramid' (raw turns, typed records, topic tracks, profiles) and lets the agent take explicit memory-navigation actions over it. **Why it matters:** Reframes long-term memory as an action space the agent controls - key for personalized assistants.
- **Task Decomposition-Guided Reranking for Adaptive Agent Skill Retrieval** - arXiv cs.AI · https://arxiv.org/abs/2607.06283
  As agent skill libraries grow, picking the right skill from many semantically similar candidates gets hard, and existing methods ignore task difficulty and skill applicability. This work decomposes the task first, then reranks candidate skills adaptively to select the optimal skill set. **Why it matters:** Directly relevant to Skills-based agent architectures where accurate skill selection is the bottleneck.
- **PatchOptic for Shared-State LLM Workflows with Projected Views and Verified Structured Updates** - arXiv cs.AI · https://arxiv.org/abs/2607.05483
  Agentic workflows over shared structured state show each model call only a fragment (progressive disclosure) via grep/RAG/AST/skills. Those handle reads but don't define when a locally-proposed rewrite is valid globally; PatchOptic adds projected views plus verified structured updates so edits stay consistent with the full state. **Why it matters:** Tackles a real correctness gap in multi-step agents that mutate shared state under limited context.
- **CHARLIE: An On-Premise Multi-Agent RAG System for Evidential Reasoning in Forensic Science** - arXiv cs.AI · https://arxiv.org/abs/2607.05428
  CHARLIE is an on-premise multi-agent RAG pipeline for digital forensics, handling heterogeneous unstructured documents under traceability, confidentiality, and legal-compliance constraints. It combines local retrieval, task decomposition, structured memory, and verification, avoiding cloud dependence. **Why it matters:** A privacy-first, verifiable multi-agent RAG pattern for high-stakes regulated domains.
- **RynnWorld-Teleop: An Action-Conditioned World Model for Digital Teleoperation** - HF Papers · https://huggingface.co/papers/2607.06558
  Robot learning is bottlenecked by physical teleoperation that ties each demonstration to specific hardware. RynnWorld-Teleop replaces the real robot with a generative world model: an operator's hand-pose stream drives a robot-centric model to synthesize high-fidelity egocentric video from one reference image, with the recorded poses serving as embodiment-agnostic labels. **Why it matters:** Decouples robot data collection from physical hardware - a potential scaling unlock for embodied AI.
- **Beyond the Leaderboard: A Synthesis of Tool-Use, Planning, and Reasoning Failures in LLM Agents** - arXiv cs.AI · https://arxiv.org/abs/2607.05775
  Benchmark gains often hide recurring agent failure modes. This paper synthesizes 27 benchmark/taxonomy/audit papers (2023-2026) across 19 benchmarks into a cross-cutting taxonomy of agent limitations spanning tool use, planning, coordination, and long-horizon operation. **Why it matters:** A useful failure-mode map for anyone building or evaluating LLM agents beyond headline scores.
- **Geometry-Aware Uncertainty Coresets for Robust Visual In-Context Learning in Histopathology** - arXiv cs.AI · https://arxiv.org/abs/2605.18419
  Fine-tuning billion-parameter VLMs on scarce pathology data is prohibitive, while in-context learning is fragile to which demonstration examples are chosen. This work builds geometry-aware, uncertainty-based coresets to pick robust ICL exemplars for histopathology VLMs without parameter updates. **Why it matters:** Improves reliability of few-shot VLM diagnostics where labeled data is expensive.
- **Danus: Orchestrating Mathematical Reasoning Agents with Fact-Graph Memory** - arXiv cs.CL · https://arxiv.org/abs/2607.06447
  Research-level math agents struggle to coordinate parallel proof search while keeping intermediate claims organized and reliable. Danus centers orchestration on a shared fact graph as global memory, tracking claims across agents to scale proof search. **Why it matters:** Shows structured shared memory as the coordination substrate for multi-agent reasoning.
- **CurateEvo: Data-Curation Evolving for Agentic Post-Training** - arXiv cs.CL · https://arxiv.org/abs/2607.06140
  Agentic post-training pipelines usually treat data curation as a fixed preprocessing step focused on augmentation. CurateEvo makes curation a failure-driven, evolving strategy represented as executable code that iteratively rewrites itself based on downstream agent failures. **Why it matters:** Closes the loop between agent failures and the training data that fixes them.
- **From Voting to Agent Collaboration: Answer-Type-Aware LLM Pipelines for BioASQ 14b** - arXiv cs.CL · https://arxiv.org/abs/2607.06452
  Instead of one prompting strategy for all biomedical questions, this framework routes yes/no, factoid, and list questions through different inference procedures and shifts from simple voting to agent collaboration for evidence integration. **Why it matters:** Answer-type-aware routing is a practical recipe for more robust domain QA.
- **PolyJarvis: An LLM-Orchestrated Agent for Automated All-Atom Molecular Dynamics of Amorphous Homopolymers** - arXiv cs.CL · https://arxiv.org/abs/2604.02537
  Running polymer MD simulations demands expertise in force fields, system construction, and equilibration. PolyJarvis couples an LLM with EMC and LAMMPS toolkits through Model Context Protocol (MCP) servers for end-to-end polymer property prediction from a molecular description. **Why it matters:** A clean example of MCP wiring an LLM to real scientific simulation tools end-to-end.
- **BabyVision: Visual Reasoning Beyond Language** - arXiv cs.CL · https://arxiv.org/abs/2601.06521
  State-of-the-art multimodal LLMs lean on linguistic priors to mask fragile visual understanding, failing basic tasks a 3-year-old solves. BabyVision is a benchmark isolating core visual abilities from linguistic knowledge to expose this gap. **Why it matters:** Quantifies how much MLLM 'vision' is actually language, guiding where perception needs work.
- **Akashic: A Low-Overhead LLM Inference Service with MemAttention** - arXiv cs.AI · https://arxiv.org/abs/2607.05708
  Agent systems accumulate context across turns, tools, and sessions, and replaying full history per request blows up prefill cost and buries relevant evidence. Akashic's MemAttention organizes context into bounded chunks and models semantic relationships across them for cheaper, higher-quality serving. **Why it matters:** An inference-layer answer to ever-growing agent context without dropping relevant history.

## 📄 Research Papers — Top Picks
- **Learning While Acting: A Skill-Enhanced Test-Time Co-Evolution Framework for Online Lifelong Learning Agents** — Bo Mao et al. · https://arxiv.org/abs/2606.04815
  LifeSkill is a two-stage RL framework letting LLM agents keep learning at test time instead of relying on static skill retrieval. Verifier-Guided Skill Learning rewards candidate skills by rollout success, while Online Skill Internalization folds skill-conditioned trajectories back into the model's parameters, avoiding context bloat. On LifelongAgentBench it adds ~7 absolute points over lifelong-agent baselines. **Why it matters:** Moves lifelong agents from external memory retrieval toward genuine in-weights internalization of new skills during deployment.
- **A Multi-Agent Audit Framework for High-Stakes Reasoning: Clinical Mental Health Screening** — Jingchen Ye, Yanpei Yu, Luyao Zhang · https://arxiv.org/abs/2606.21123
  Decomposes high-stakes reasoning into a Perception Agent, RAG, chain-of-thought inference, and a dedicated Audit verification stage via a modular LangChain workflow. On DAIC-WOZ it cuts PHQ-8 depression-severity MAE from 5.35 to 5.02 over single-agent baselines while exposing cross-agent validation traces. **Why it matters:** Shows multi-agent verification + audit trails can improve both accuracy and interpretability where hallucination is unacceptable.
- **Context-Aware Generative AI for Automated Telecom Test Script Generation** — Gautam Prasad, Chandramohan T. N., Joy Bose · https://arxiv.org/abs/2606.21151
  Treats testing as a continuously adapting process: a live knowledge graph is the single source of truth, a delta engine detects fine-grained system changes, and an MCP-operating generative agent creates, updates, or retires only the affected tests. RAG enriches reasoning with telecom-domain knowledge. **Why it matters:** Concrete MCP-based agent design for keeping test suites in sync with evolving systems instead of regenerating them wholesale.
- **Decoupling the Declarative from the Procedural in Vision-Language-Action Models** — Nikolaos Tsagkas et al. · https://arxiv.org/abs/2606.21496
  w2VLA restructures VLA information flow so robot-state sequences are modulated compositionally by visual, spatial, and skill signals rather than dumped into one opaque action transformer. This decouples 'what things are' from 'how to act', enabling robust behavior cloning and strong zero-shot skill transfer to unseen objects. **Why it matters:** Targets the brittleness bottleneck in robot foundation models — generalizing skills to novel objects without per-object data collection.
- **Anything2Skill: Compiling External Knowledge into Reusable Skills for Agents** — Qianjun Pan et al. · https://arxiv.org/abs/2606.09316
  A taxonomy-guided framework that compiles heterogeneous knowledge (manuals, logs, trajectories) into structured, executable skill contracts stored in a versioned SkillBank. At inference, agents retrieve both declarative passages (RAG) and procedural skills, hitting 98.85% and 94.10% success on qsv and GitHub-CLI versus RAG-only. **Why it matters:** Extends RAG from knowledge access to capability reuse by turning latent procedures into explicit, retrievable agent skills.
- **CLI-Universe: Towards Verifiable Task Synthesis Engine for Terminal Agents** — Zhanbo Hua et al. (HF Papers) · https://huggingface.co/papers/2606.22883
  A synthesis engine that generates terminal-agent tasks across a capability taxonomy, grounds them in real technical materials, then validates each in Dockerized environments with strict fail-to-pass checks (discarding ~2/3 of candidates). Fine-tuning Qwen3-32B on the resulting CLI-Universe-6K hits 33.4% on Terminal-Bench 2.0, SOTA for open-data models <=32B. **Why it matters:** Shows rigorous executable verification of synthetic data beats scale, closing the high-quality training-data gap for terminal agents.
- **VADAOrchestra: Neurosymbolic Orchestration of Adaptive Reasoning Workflows** — Teodoro Baldazzi et al. · https://arxiv.org/abs/2606.22485
  Combines LLM flexibility with symbolic rigor: an LLM orchestrator incrementally plans a workflow encoded as a Datalog+/- logic program (predicates = tool calls), executed by a symbolic engine for verifiable, auditable traces. Decoupling orchestration from inference also addresses scalability on large datasets. **Why it matters:** A neurosymbolic recipe for agent workflows that need auditability and reproducibility, e.g. regulated financial decisions.
- **VideoAgent: All-in-One Framework for Video Understanding and Editing** — Hengji Zhou et al. · https://arxiv.org/abs/2606.23327
  An agentic framework with shot-planning agents and 30+ specialized editing agents orchestrated via intent parsing and textual-gradient graph optimization. It reaches 87-95% orchestration success while cutting API costs 60%, with human ratings only 4% below human-made videos. **Why it matters:** Demonstrates large multi-agent tool orchestration producing near-professional creative output at sharply lower cost.
- **Less is More: Lightweight Prompt Compression for QA on Edge Devices** — Zihuai Xu et al. · https://arxiv.org/abs/2606.20571
  CORE is a two-stage, SLM-free sentence-level prompt compressor: it builds answer/clue sets via NER and semantic matching, then refines with orthogonal residual retrieval and spatial-proximity filtering. On Jetson Orin it improves accuracy >=30% within a 2000-token budget while halving memory; on a smartphone it cuts energy 95.74% vs LLMLingua2. **Why it matters:** Makes agentic RAG viable on resource-constrained edge hardware by compressing context without an auxiliary model.
- **StatABench: Evaluating Statistical Analysis Capabilities of LLMs** — Youxin Zhu et al. · https://arxiv.org/abs/2606.22977
  A benchmark with 404 closed-form questions across 18 statistical topics plus 30 open-ended modeling tasks, evaluated with a LangChain MCP framework and data-science agents. Even GPT-5.1 reaches only 68.6% on the closed set; the best open model 60.6%. **Why it matters:** Exposes a persistent gap in tool-grounded statistical reasoning that current frontier and open models have not closed.
- **PlanBench-XL: Evaluating Long-Horizon Planning of LLM Tool-Use Agents in Large Tool Ecosystems** — Jiayu Liu et al. · https://arxiv.org/abs/2606.22388
  An interactive benchmark of 327 retail tasks over 1,665 tools that tests retrieval-limited tool visibility plus an optional blocking mechanism simulating missing/failing/distracting tools. GPT-5.4 scores 51.90% block-free but collapses to 11.36% under severe blocking. **Why it matters:** Quantifies how fragile agent planning becomes in large, imperfect tool ecosystems — the realistic deployment setting.
- **VISTA Architect: A graph database-oriented health AI system for tumor boards** — Tuomo Kiiskinen et al. · https://arxiv.org/abs/2606.22692
  Transforms longitudinal EHRs into a provenance-linked knowledge graph at ingestion, with an abstracted timeline layer built by graph-guided LLM extraction, avoiding repeated raw-text processing. Across 1,180 Stanford patients it hits 96.4% accuracy on tumor-board variables and cuts 30-patient prep to ~2.2 minutes. **Why it matters:** A precompute-once knowledge-graph pattern that beats long-context prompting and RAG on cost, latency, and temporal accuracy.
- **Are LLMs Effective Negotiators? Systematic Evaluation in Negotiation Dialogues** — Deuksin Kwon et al. · https://arxiv.org/abs/2402.13550
  Systematically evaluates LLMs across the stages of negotiation — context comprehension, Theory-of-Mind, strategic reasoning, and communication. GPT-4 leads on many tasks but struggles with subjective assessment and generating strategically advantageous, context-appropriate responses. **Why it matters:** Maps where negotiation agents still fail, guiding design of dialogue systems and pedagogical feedback tools.
- **From RAG to Agentic RAG for Faithful Islamic Question Answering** — Gagan Bhatia et al. · https://arxiv.org/abs/2601.07528
  Introduces IslamicFaithQA, a 3,810-item bilingual generative benchmark measuring hallucination and abstention, plus an agentic RAG framework using structured tool calls for iterative evidence seeking and answer revision. Agentic RAG yields the largest gains beyond standard RAG, reaching SOTA even with a small Qwen3-4B. **Why it matters:** Shows agentic, tool-calling retrieval improves faithfulness and abstention in high-consequence grounded QA.
- **Hypothesis-Driven Skill Optimization for LLM Agents** — Fangxin Shang, Yehui Yang · https://arxiv.org/abs/2606.22330
  HDSO is a train-free framework where a frozen curator proposes falsifiable hypotheses about useful skills, validates them via paired control/treatment executions, and consolidates only supported skills into a repository the executor consumes via progressive disclosure. On ALFWorld it adds +6.9 (Qwen3-8B) and stays +7.1 even under 20% flipped feedback. **Why it matters:** An auditable, noise-robust skill lifecycle that avoids the spurious-shortcut risk of unconstrained agent memory accumulation.

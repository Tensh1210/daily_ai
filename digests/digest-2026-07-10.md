## 📄 Research Papers — Top Picks

- **UniClawBench: A Universal Benchmark for Proactive Agents on Real-World Tasks** — Chen, Duan, Sun et al. · https://huggingface.co/papers/2607.08768
  UniClawBench is the first capability-driven benchmark that evaluates proactive agents in live Docker containers across five foundational skills: Skill Usage, Exploration, Long-Context Reasoning, Multimodal Understanding, and Cross-Platform Coordination. It uses 400 bilingual real-world tasks with fine-grained step checkpoints and a closed-loop executor/supervisor/user-agent evaluation instead of static answers. This design disentangles base-model capability from agent-framework design. **Why it matters:** Gives the field a rigorous, contamination-resistant way to measure real-world agent competence rather than sandbox toy tasks.

- **WebSwarm: Recursive Multi-Agent Orchestration for Deep-and-Wide Web Search** — Song, Zhang, Zhao et al. · https://arxiv.org/abs/2607.08662
  WebSwarm is a progressive recursive-delegation framework where agentic search nodes each pair a local objective with a search mode, and can solve or further delegate to child nodes, returning evidence upward for aggregation. It probes how web information is organized before expanding and reuses process-level experience across sibling nodes. It beats single- and multi-agent baselines on deep, wide, and interleaved search benchmarks. **Why it matters:** Recursive delegation addresses the depth-vs-coverage tradeoff that constrains single-trajectory ReAct search agents.

- **DeepSearch-World: Self-Distillation for Deep Search Agents in a Verifiable Environment** — Geng, He, Chen et al. · https://arxiv.org/abs/2607.07820
  DeepSearch-Evolve trains web agents via self-distillation inside DeepSearch-World, a deterministic, reproducible environment with 420K multi-hop QA tasks and verifiable search/read tools. It iteratively generates, filters, mixes, and fine-tunes trajectories, supporting progress verification, grounded reflection, and failure recovery. Without distilling from stronger models, the 9B agent hits 31.2% BrowseComp, 61.5% GAIA, and 93.4% HotpotQA. **Why it matters:** Shows verifiable environments enable scalable self-improvement for long-horizon agents without a stronger teacher.

- **Token-Flow Firewall: Semantic Runtime Auditing for Persistent AI Agents** — Wang, Zhang, Zhang et al. · https://arxiv.org/abs/2607.08395
  TokenWall is a runtime defense that treats natural-language token flows (memory updates, tool arguments, retrieved files, inter-component messages) as the security boundary for persistent agents. It performs boundary-aware semantic auditing with lightweight local inspection and selective escalation of ambiguous high-risk cases. On CIK-Bench it cuts attack success to 12.5% while keeping a 97.4% benign pass rate and adding only 0.69s latency. **Why it matters:** Persistent agents create a large semantic attack surface; pre-execution flow mediation is a practical containment layer.

- **Cognitive-structured Multimodal Agent for Understanding, Generation, and Editing** — Wang, Fu, Huang et al. · https://arxiv.org/abs/2607.08497
  This agent externalizes visual inputs into an Episodic Visual Memory and selectively reactivates relevant episodes, avoiding the visual-token explosion of feeding all history into one context window. It combines a perceptual abstraction engine, cross-turn retrieval engine, and executive controller, trained with RL on programmatically generated multi-turn data. The 8B model reaches 91.4% retrieval accuracy over 20-turn sessions, beating 32B baselines by +8.2% while nearly halving per-turn latency. **Why it matters:** Structured episodic memory scales long-horizon multimodal dialogue better than raw parameter growth.

- **DR-Arena: an Automated Evaluation Framework for Deep Research Agents** — Gao, Zhao, Deng, Zhang · https://arxiv.org/abs/2601.10504
  DR-Arena is a fully automated framework that evaluates Deep Research agents via dynamic investigation, building real-time Information Trees from fresh web trends to avoid contamination and temporal misalignment. An automated Examiner generates tasks testing deep reasoning and wide coverage, and an Adaptive Evolvement Loop escalates complexity until a capability boundary emerges. It achieves 0.94 Spearman correlation with the LMSYS Search Arena leaderboard with no manual effort. **Why it matters:** Offers a self-updating, human-preference-aligned alternative to static, quickly-stale research-agent benchmarks.

- **DeepTutor: Towards Agentic Personalized Tutoring** — Zhao, Zhang, Ren et al. · https://arxiv.org/abs/2604.26962
  DeepTutor is an open-source agentic framework unifying citation-grounded tutoring with difficulty-calibrated question generation, using a hybrid engine that couples static knowledge grounding with dynamic learner memory. It introduces TutorBench, an interactive benchmark with profile-driven student simulators across five university-level domains. It improves personalized metrics by 10.8% and general agentic reasoning by 29.4% across five backbones. **Why it matters:** Concrete blueprint for personalized, memory-adaptive tutoring agents beyond static RAG.

- **Narration-of-Thought: Inference-Time Scaffolding for Defeasible Ethical Reasoning** — Cooper, Velasquez · https://arxiv.org/abs/2606.26366
  NoT is a training-free system prompt that structures chain-of-thought into five sections — protagonist, stakeholders, two-step consequences, uncertainty, then commitment. On 100 DailyDilemmas scenarios it cuts "stakeholder collapse" from up to 31% to under 1% and uncertainty suppression from up to 72% to 1–24%, with a matched-budget control ruling out mere token spend. Extended to a debate protocol it converts a 6% standoff into 95–100% consensus. **Why it matters:** A zero-cost prompt scaffold produces auditable moral reasoning traces for dependable agentic deployment.

- **Who Broke the System? Failure Localization in LLM-Based Multi-Agent Systems** — Xia, Gao, Quan et al. · https://arxiv.org/abs/2607.07989
  AgentLocate attributes multi-agent failures to both a specific agent and the earliest decisive step, combining an LLM judge with multi-perspective verification aggregated by a confidence-aware strategy. Feedback is used to adapt the judge via lightweight fine-tuning. It outperforms existing failure-localization methods on two benchmarks while staying efficient in tokens and runtime. **Why it matters:** Diagnosing which agent and when a trajectory went wrong is essential infrastructure for debugging production multi-agent systems.

- **CausalDS: Benchmarking Causal Reasoning in Data-Science Agents** — Leban, Sun · https://huggingface.co/papers/2607.08093
  CausalDS evaluates causal reasoning in agentic data-science workflows using synthetically generated structural causal models paired with realistic natural-language stories, reducing "causal parrot" contamination. Tasks span all three of Pearl's rungs and require multi-tool coding over imperfect observations, and knowing when to abstain is a first-class scored outcome. It jointly measures symbolic causal reasoning, data science, uncertainty, abstention, and tool use. **Why it matters:** Fills the gap between symbolic causal benchmarks and realistic data-analysis benchmarks for tool-using agents.

- **From Solvers to Research: LLM-Driven Formal Mathematics at the Research Frontier** — Jiang, Liang, Zhang et al. · https://arxiv.org/abs/2607.07779
  This position paper reviews AI4Math — datasets, auto-formalization, proof synthesis — and argues current LLM theorem provers succeed on well-defined problems but fail at open-ended frontier research math. It calls for a shift from predefined problem-solvers to research agents that can discover theorems and resolve open conjectures with rigorous formal reasoning. It maps limitations across data, relational structure, exploration, tooling, and human-AI collaboration. **Why it matters:** Frames the roadmap for turning provers into genuine mathematical research agents.

- **Tool-Making and Self-Evolving LLM Agents in Low-Latency Systems** — Kujanpää, Liu, Alam et al. · https://arxiv.org/abs/2607.08010
  This agentic pipeline compiles repeated SOP steps into validated, versioned tools before deployment, grounding synthesis in the live environment via execution traces and schema observation, then repairing against labeled cases. At runtime the agent calls tools directly and only falls back to code generation when needed. In a fulfillment-center alarm-triage deployment, tool calls cut p50 latency 42% and end-to-end error up to 53%. **Why it matters:** Turning recurring agent code into cached tools makes industrial LLM systems faster, more reliable, and auditable.

- **MASTE: A Multi-Agent Pipeline for Zero-Shot Aspect Sentiment Triplet Extraction** — Hong, Wang, Yue et al. · https://arxiv.org/abs/2607.08080
  MASTE decomposes aspect-sentiment triplet extraction into four sequential stages handled by specialized agents that condition on prior outputs, avoiding the single-decoding-step bottleneck of one-pass LLM extraction. It is entirely training-free and generalizes across backbones and datasets. On four ASTE benchmarks it substantially beats zero-shot and chain-of-thought baselines, narrowing the gap to supervised methods without labeled triplets. **Why it matters:** Shows agentic task decomposition can rival supervised systems on structured extraction with no training data.

- **From Prompts to Contracts: Harness Engineering for Auditable Enterprise LLM Agents** — Ahn, Kim · https://arxiv.org/abs/2607.08028
  This work moves deterministic agent behavior — source boundaries, entity routing, answer contracts, reproducible traces — out of prompts and into code, manifests, schemas, and validation artifacts around a replaceable model boundary. Across three hosted models, contract checks passed on all 270 composition-boundary runs, and code-owned enforcement preserved full utility (120/120) where a bolt-on guardrail over-refused (88/120). Prompt instructions alone let trace-leakage and recommendation-language violations reach the reader. **Why it matters:** A reusable pattern for making enterprise agents auditable and safe without sacrificing utility to prompt-only guardrails.

- **ICDAR 2026 HIPE-OCRepair Competition on LLM-Assisted OCR Post-Correction** — Ehrmann, Boros, Opitz et al. · https://arxiv.org/abs/2607.08143
  HIPE-OCRepair-2026 evaluates modern LLM-assisted OCR post-correction of historical newspapers and printed works in English, French, and German (17th–20th century), working at paragraph/article level without source images. Four teams submitted systems from zero-shot prompting to continued pre-training and fine-tuning on a harmonized multilingual dataset. Results show significant OCR gains but recurring over-correction on low-noise inputs. **Why it matters:** Provides a reproducible benchmark and shows both promise and hallucination risks of LLMs for digital-heritage restoration.

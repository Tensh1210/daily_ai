# AI Daily Digest — 2026-06-25

## 🚀 Product & Breakthroughs
- **Introducing computer use in Gemini 3.5 Flash** — Google DeepMind · https://deepmind.google/blog/introducing-computer-use-in-gemini-3-5-flash/
  DeepMind brings a "computer use" capability to its fast, low-cost Gemini 3.5 Flash model, letting it perceive and act on a GUI (clicking, typing, navigating) to complete tasks autonomously. Putting agentic UI control in the cheap/fast tier signals computer-use agents are moving from frontier demos toward routine, high-throughput deployment. **Why it matters:** Cheap GUI-driving agents make automated workflows over existing software economically viable at scale.
- **OpenAI and Broadcom unveil LLM-optimized inference chip** — OpenAI News · https://openai.com/index/openai-broadcom-jalapeno-inference-chip
  OpenAI and Broadcom introduced "Jalapeño," custom silicon designed specifically for LLM inference to improve performance, efficiency, and scale. It marks OpenAI's deeper push into vertically-integrated hardware to reduce reliance on general-purpose GPUs. **Why it matters:** Purpose-built inference chips could materially cut serving costs and tighten the model-hardware co-design loop.

## 📄 Research Papers — Top Picks
- **The Hitchhiker's Guide to Agentic AI: From Foundations to Systems** · https://arxiv.org/abs/2606.24937
  A comprehensive practitioner's reference covering the full agentic-AI stack: the LLM substrate (transformers, training/fine-tuning, SFT/LoRA/MoE, compression, inference) up through production agent system design. Frames great agentic systems as requiring understanding of every pipeline layer, not just prompting. **Why it matters:** A consolidated end-to-end reference lowers the barrier for teams building production agents.
- **Verifiable Manifest Signing and Transparency Enforcement for Secure MCP-Based LLM Pipelines** · https://arxiv.org/abs/2601.23132
  MCP standardizes tool exposure/invocation but does not require tool manifests to be cryptographically signed, freshness-checked, or auditable before execution, leaving pipelines open to manifest tampering. The paper proposes verifiable signing plus transparency enforcement to authenticate and audit tool manifests before they run. **Why it matters:** Supply-chain integrity for MCP tools is a prerequisite for safely deploying agents in regulated domains.
- **Skill-MAS: Evolving Meta-Skill for Automatic Multi-Agent Systems** · https://arxiv.org/abs/2606.18837
  Tackles the dilemma where inference-time multi-agent systems use strong frozen LLMs but never learn, while training-time systems learn but are capped by weak small models. Skill-MAS evolves reusable "meta-skills" to retain experience while still leveraging frontier LLMs. **Why it matters:** Lets multi-agent systems accumulate experience without retraining the underlying frontier model.
- **Why Multi-Step Tool-Use Reinforcement Learning Collapses and How Supervisory Signals Fix It** · https://arxiv.org/abs/2606.26027
  Finds that RL alone on multi-step tool-use tasks can cause catastrophic collapse — performance abruptly drops as tool-invocation structure fails — traced to probability spikes in specific control tokens. Adding supervisory signals stabilizes training and restores structured execution. **Why it matters:** Identifies a concrete failure mode and fix for training reliable tool-using agents with RL.
- **Beyond Function Calling: Benchmarking Tool-Using Agents under Tool-Environment Unreliability** · https://arxiv.org/abs/2606.25819
  Existing tool-use benchmarks assume clean, trustworthy environments; ToolBench-X instead evaluates agents under recoverable reliability hazards across sequential, parallel, and mixed multi-step workflows. Exposes how brittle current agents are when tools fail or behave unexpectedly. **Why it matters:** Real deployments face flaky tools, so robustness benchmarks are needed to measure production-readiness.
- **Quantization Inflates Reasoning: Token Inflation as a Hidden Cost of Low-Bit Reasoning Models** · https://arxiv.org/abs/2606.25519
  Shows INT4/INT3 post-training quantization can preserve final-answer accuracy on reasoning and agentic tool-use tasks while silently inflating chain-of-thought length — a hidden test-time compute cost not captured by accuracy or per-token latency. **Why it matters:** Quantization savings may be partly eaten by longer reasoning traces, complicating cost estimates for low-bit reasoning models.
- **V-Zero: Answer-Label-Free On-Policy Distillation for Fine-Grained Visual Reasoning** · https://huggingface.co/papers/2606.25319
  Proposes on-policy distillation with contrastive evidence gating so multimodal LLMs learn fine-grained visual reasoning from their own sampled trajectories, avoiding costly RL rewards, hand-designed verification rules, or large annotated reasoning traces. **Why it matters:** Cuts the supervision cost of training models that must ground reasoning in specific image regions.
- **MemDreamer: Hierarchical Graph Memory and Agentic Retrieval for Long Video Understanding** · https://arxiv.org/abs/2606.07512
  Decouples perception from reasoning to handle hours-long videos, streaming frames into a three-tier Hierarchical Graph Memory and using agentic retrieval instead of processing full visual sequences (which causes token explosion and attention dilution). Plug-and-play framework. **Why it matters:** Makes long-video understanding tractable by turning it into an agentic memory-retrieval problem.
- **AgentOdyssey: Open-Ended Long-Horizon Text Game Generation for Test-Time Continual Learning Agents** · https://arxiv.org/abs/2606.24893
  Procedurally generates open-ended text games with rich entities and long-horizon tasks to evaluate agents on exploration, knowledge acquisition, episodic retention, and long-horizon planning at test time. Goes beyond fixed-distribution ML assumptions. **Why it matters:** Provides a benchmark for the continual, test-time learning that real-world agents need.
- **BitNet Text Embeddings** · https://arxiv.org/abs/2606.25674
  BITEMBED converts pretrained LLM backbones into BitNet-style extreme low-bit embedding encoders, jointly cutting encoding compute and the storage/bandwidth cost of high-dimensional full-precision vectors in large indexes. **Why it matters:** Low-bit embeddings could sharply reduce retrieval/RAG infrastructure cost at scale.
- **To Isolate or to Score? Model-Adaptive Assessment for Cost-Efficient Multi-Agent RAG** · https://arxiv.org/abs/2606.25191
  Controlled study of training-free document-assessment interventions on 7B–9B models for RAG, finding a sharp dichotomy: weaker models benefit most from per-document isolation (matching full multi-agent assessment), while stronger ones need scoring. **Why it matters:** Guides picking the cheapest assessment mechanism per model size in multi-agent RAG.
- **ASAP: Agent-System Co-Design for Wall-Clock-Centered Auto HPO** · https://arxiv.org/abs/2606.25207
  Reframes LLM-driven hyperparameter optimization around wall-clock efficiency rather than per-iteration gains, co-designing the agent and system since every HPO tool's surrogate prior carries inductive bias that fails on diverse problems. **Why it matters:** Wall-clock-centered HPO better reflects real research budgets than per-iteration metrics.
- **Reinforcement Learning Improves Traversal of Parametric Knowledge in LLMs** · https://arxiv.org/abs/2511.05933
  Challenges the "RL improves reasoning at the cost of knowledge" narrative: reasoning models outperform their instruction-tuned versions even on pure knowledge recall, reflecting improved procedural skill in navigating existing parametric knowledge hierarchies rather than newly acquired facts. **Why it matters:** Suggests RL sharpens knowledge retrieval, not just reasoning, reframing how we evaluate it.
- **daVinci-kernel: Co-Evolving Skill Selection, Summarization, and Utilization via RL for GPU Kernel Optimization** · https://arxiv.org/abs/2606.16497
  An RL framework where three agents sharing one LLM backbone (skill selection via BM25+LLM reranking, a multi-turn CUDA/Triton policy, and skill summarization) co-evolve a dynamic skill library for GPU kernel optimization, assuming correctness and optimizing execution efficiency. **Why it matters:** Demonstrates evolving skill libraries can automate low-level performance engineering.
- **Failure Modes of LLMs on Research-Level Mathematics: A Taxonomy and Empirical Characterisation** · https://arxiv.org/abs/2606.24902
  Analyzes why top LLMs are confidently wrong on research-level math, identifying four failure modes — citation fabrication, premise smuggling, silent problem reformulation, and local-to-global compatibility gaps — via audits of one-shot proofs. **Why it matters:** Names concrete, recurring reasoning failures to target when pushing LLMs toward genuine mathematical research.

## 📄 Research Papers - Top Picks

- **Beyond Static Leaderboards: Predictive Validity for the Evaluation of LLM Agents** - arXiv cs.AI · https://arxiv.org/abs/2606.19704
  Aggregates the largest coordinated deep-dive of an MCP-based industrial-agent benchmark to date - fourteen parallel implementation studies spanning new asset classes including a multi-modal visual extension. Argues no single benchmark covers more than four or five of the dimensions deployment actually exposes, and pushes toward predictive validity over static leaderboard rank. **Why it matters:** Reframes agent evaluation around real deployment dimensions, directly relevant to anyone building MCP/tool-using agents.
- **Navigating Unreliable Parametric and Contextual Knowledge: Explicit Knowledge Conflict Resolution for LLM Inference** - arXiv cs.AI · https://arxiv.org/abs/2606.20245
  Tackles the conflict that arises when an LLM's parametric memory disagrees with retrieved or in-context evidence. Proposes explicit conflict-resolution at inference rather than blindly trusting either source, improving robustness when external knowledge is noisy or contradictory. **Why it matters:** Conflict-aware inference is core to making RAG and tool-augmented systems trustworthy.
- **MedRLM: Recursive Multimodal Health Intelligence for Long-Context Clinical Reasoning** - arXiv cs.CL · https://arxiv.org/abs/2606.20164
  A medical model that reasons over heterogeneous, longitudinal patient data instead of isolated QA, combining sensor-guided screening, evidence-grounded decision support and referral optimization. Uses recursive multimodal reasoning to stay robust when clinical evidence is dispersed across long contexts. **Why it matters:** Shows long-context plus multimodal RAG patterns applied to high-stakes clinical decision support.
- **Human-on-the-Loop Orchestration for AI-Assisted Legal Discovery** - arXiv cs.AI · https://arxiv.org/abs/2606.19812
  Studies failure modes of autonomous LLM agents in e-discovery, where compounding multi-step errors can become legal malpractice. Names 'trajectory collapse' over privileged document corpora and introduces human-on-the-loop orchestration to contain it. **Why it matters:** Concrete pattern for keeping high-risk agentic workflows safe with human oversight.
- **DynAMO: Dynamic Asset Management Orchestration via Topological Multi-Agent Scheduling** - arXiv cs.AI · https://arxiv.org/abs/2606.19382
  A deployment-ready Plan-then-Execute engine for industrial asset lifecycles that generates verifiable workflow graphs. Targets the latency, concurrency-instability and safety risks that block real Industry 4.0 agent deployments. **Why it matters:** Production-oriented multi-agent orchestration with verifiability baked in.
- **AtomMem: Building Simple and Effective Memory System for LLM Agents via Atomic Facts** - arXiv cs.CL · https://arxiv.org/abs/2606.19847
  Proposes a memory system for agents built from atomic facts rather than coarse, unstable memory chunks, enabling stable long-term accumulation and reuse across multi-session interactions. Aims to fix the fragility of existing memory-augmented setups under fixed context windows. **Why it matters:** Memory is the bottleneck for persistent agents; an atomic-fact design is a clean, reusable primitive.
- **ORAgentBench: Can LLM Agents Solve Challenging Operations Research Tasks End to End?** - arXiv cs.AI · https://arxiv.org/abs/2606.19787
  Benchmarks LLM agents on realistic operations-research work end-to-end - from problem to solved instance - rather than decoupling modeling from solving on pre-formalized inputs. Exposes how far current agents are from full OR workflows in executable environments. **Why it matters:** A rigorous end-to-end agent benchmark in a domain with verifiable ground truth.
- **ShoppingBench: A Real-World Intent-Grounded Shopping Benchmark for LLM-based Agents** - arXiv cs.CL · https://arxiv.org/abs/2508.04266
  An end-to-end shopping benchmark grounded in complex real-world intents - applying vouchers, managing budgets, multi-product sellers - beyond simple find/buy tasks. Tests whether agents can execute multi-constraint goals in a realistic e-commerce environment. **Why it matters:** Pushes agent benchmarks toward messy, multi-constraint user goals that mirror production.
- **Hierarchical Control in Multi-Agent Games: LLM-based Planning and RL Execution** - arXiv cs.AI · https://arxiv.org/abs/2606.20014
  Pairs a pretrained LLM as a high-level planner with RL for low-level execution in complex multi-agent environments. The hierarchy addresses sparse rewards and huge state-action spaces where pure RL struggles to learn coordination. **Why it matters:** A practical LLM-plans / RL-acts split for hard multi-agent coordination.
- **FM-Agent: Scaling Formal Methods to Large Systems via LLM-Based Hoare-Style Reasoning** - arXiv cs.AI · https://arxiv.org/abs/2604.11556
  Uses LLM-driven Hoare-logic reasoning to verify large generated systems by decomposing correctness arguments. Targets the gap where LLMs can now generate compiler-scale code but its correctness is hard to establish. **Why it matters:** Bridges LLM code generation with formal verification at system scale.
- **FundaPod: A Multi-Persona Agent Pod Platform with Knowledge Graph Memory for Fundamental Investment Research** - arXiv cs.AI · https://arxiv.org/abs/2605.27864
  A multi-persona agent platform backed by knowledge-graph memory for institutional fundamental research - gathering evidence, comparing viewpoints and generating investment theses rather than just predicting signals. Models the analyst workflow as cooperating personas. **Why it matters:** Shows KG-backed memory plus role specialization for evidence-heavy research agents.
- **Trustworthy Multi-Agent Systems: Mitigating Semantic Drift with the Argent Signaling Protocol** - arXiv cs.CL · https://arxiv.org/abs/2606.19356
  Distinguishes grounded-but-incomplete answers from ungrounded ones in multi-agent LLM systems, instead of treating all failures with blind retries. The Argent signaling protocol lets human supervisors tell whether a retry is warranted. **Why it matters:** Failure-type signaling makes multi-agent retries and human oversight far more efficient.
- **Multi-Agent Transactive Memory** - arXiv cs.CL · https://arxiv.org/abs/2606.19911
  Proposes infrastructure for knowledge sharing across heterogeneous agent populations - retrieval systems that index agent-generated artifacts for reuse, analogous to how search engines index human artifacts. Enables decentralized agents with diverse skills to build on each other's work. **Why it matters:** A shared-memory substrate for ecosystems of specialized agents.
- **HydraHead: From Head-Level Functional Heterogeneity to Specialized Attention Hybridization** - arXiv cs.CL · https://arxiv.org/abs/2606.20097
  Revisits hybrid attention by mixing linear and full attention at the head level rather than the usual layer-wise split, exploiting functional heterogeneity across heads. Aims to tame the quadratic cost of long-context attention without the integration pitfalls of layer-wise hybrids. **Why it matters:** A finer-grained efficiency lever for long-context models.
- **PerceptionDLM: Parallel Region Perception with Multimodal Diffusion Language Models** - arXiv cs.CL · https://arxiv.org/abs/2606.19534
  A multimodal diffusion language model optimized for perception tasks that require captioning many regions, replacing slow autoregressive generation with parallel decoding. Targets efficiency gains for dense visual understanding. **Why it matters:** Diffusion-style parallel decoding as a speed path for multimodal perception.

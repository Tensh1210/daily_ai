## 📄 Research Papers — Top Picks
- **Euclid-MCP: A Model Context Protocol Server for Deterministic Logical Reasoning via Prolog** — Bartolomeo Bogliolo · https://arxiv.org/abs/2607.21412
  Euclid-MCP is an open-source MCP server that gives LLM agents deterministic logical reasoning through SWI-Prolog. It introduces Euclid-IR, an engine-agnostic intermediate representation for Horn-clause logic that LLMs can easily generate and compile to Prolog, exposed via a translate-run-inspect-repair tool loop. On an IT security/compliance case, LLMs alone hallucinate on larger knowledge bases while Euclid-MCP returns exact answers with lower latency. **Why it matters:** A standardized MCP interface for symbolic reasoning gives agents a reliable substrate for rule enforcement where RAG fundamentally fails.

- **DynamicMCPBench: A Trace-Grounded, Effect-Scored Benchmark for LLM Agents over Live MCP Servers** — Kamiński et al. · https://arxiv.org/abs/2607.20531
  DynamicMCPBench is a reusable framework (not a fixed dataset) for evaluating LLM agents against live, stateful MCP servers, scoring effect checkpoints rather than final answers. Run at scale (24 models, 121 servers, 750 tasks), even the strongest agents solve only ~half; 31% of tasks are solved by no model, and accuracy collapses from 39% to 13% as tool chains lengthen. Human validation confirms the automatic scoring is reliable. **Why it matters:** Exposes a consistent, quantified failure of current agents at long multi-step MCP tasks — and lets practitioners re-run the benchmark on their own servers.

- **From Chatbot to Digital Colleague: The Paradigm Shift Toward Persistent Autonomous AI** — Zhang et al. · https://arxiv.org/abs/2606.14502
  A survey framing the shift from conversational LLMs to persistent "digital colleague" systems along two axes: a cognitive core moving from fast next-token prediction to deliberate Thinking LLMs, and task execution moving from ad-hoc tool-calling agents to workstation systems with persistent workspaces, skills, verification loops, and governance. It also traces the data shift from instruction-response pairs to State-Action-Observation trajectories, and evaluation from static benchmarks to auditable self-evolving ecosystems. **Why it matters:** Offers a coherent vocabulary for the "workspace + skill" agent paradigm many teams are now converging on.

- **Student-Centered Distillation Narrows the Agentic Gap Between Small and Large LLMs** — Lyu et al. · https://arxiv.org/abs/2509.14257
  SCoRe is a distillation framework where the student generates its own trajectories and the teacher corrects only the earliest error, producing training data matched to the student's actual weaknesses. After SFT on corrected trajectories, short-horizon RL starts from the verified prefix before that error, improving stability. A 7B student distilled with SCoRe closes the agentic gap with a 72B teacher across 12 benchmarks. **Why it matters:** Makes small, cheap models competitive on multi-step tool-use tasks without imitating full teacher traces.

- **OpenSkillRisk: Benchmarking Agent Safety When Using Real-World Risky Third-Party Skills** — Liu et al. · https://arxiv.org/abs/2607.20121
  OpenSkillRisk is a safety benchmark of 263 real risky skills scraped from public skill marketplaces, grouped into seven threat categories, each paired with a task and sandbox. Across three CLI agent frameworks and thirteen LLMs, even the safest configs still execute unsafe actions ~17% of the time, with context-dependent and system-level risks hardest to avoid. Three recurring failure patterns: failing to recognize risk, recognizing but not intervening, or exceeding user intent. **Why it matters:** Quantifies a concrete supply-chain attack surface as agents increasingly install third-party skills.

- **Verifier-First Evaluation of Agentic LLMs for Infrastructure-as-Code Generation** — Mohamed Jouini · https://arxiv.org/abs/2607.20478
  A verifier-first study of seven agentic strategies for Terraform generation on a 186-task AWS benchmark, separating failures into validate/plan/policy stages. Active retrieval via ReAct agents with MCP or RAG lifts a Qwen2.5-Coder 7B from 14% to 46% pass@1; iterative refinement with verifier feedback reaches 63% (Qwen) and 84% (GPT-4o). Reflective prompt optimization adds gains without weight updates, and 79% of remaining policy failures are resolvable when policy text is made visible. **Why it matters:** Shows verifier feedback and retrieval — not bigger models — drive most of the gains in verifiable IaC generation.

- **Skill-Contracted Agents for Evidence-Aware Materials Literature Analysis** — Li et al. · https://arxiv.org/abs/2607.20431
  AlphaAgent decouples retrieval-based QA from paper-level report generation using explicit "skill contracts": a retrieval skill rewrites requests into material-specific search intents over a 300k+ paper index and reformulates when evidence is thin, while a separate skill parses full-text PDFs into structured reports. In a blind evaluation on 40 materials-science questions, it substantially outperformed a matched baseline, especially on mechanistic explanation and credibility awareness. **Why it matters:** Demonstrates explicit task separation via skill contracts as a practical pattern for evidence-grounded domain agents.

- **Learning to Detect UI Principle Violations via Reinforcement Learning** — Mehta et al. · https://arxiv.org/abs/2607.20690
  This work trains a lightweight 4B vision-language model as a critic for generated web interfaces, unifying 19 interface-quality principles from WCAG 2.2, deceptive-design taxonomies, and HCI theory. Training data is built by synthetically injecting known violations into ~10,000 clean LLM-generated Tailwind pages; continued RL lifts micro-F1 from 36% to 84%, with 13 of 19 principles exceeding 80% F1. The critic can audit interfaces, filter training data, and provide a reward signal for design-aware code generation. **Why it matters:** Adds an affordable quality gate beyond "it compiles" for the growing volume of LLM-generated frontends.

- **DataPrep-Bench: Benchmarking LLMs as Training Data Preparators** — Liang et al. · https://arxiv.org/abs/2607.20465
  DataPrep-Bench is the first unified benchmark scoring LLMs on two data-prep capabilities — constructing supervised training data from raw sources, and predicting a dataset's downstream training value — under a shared downstream-grounded protocol across six domains. It ships a skill-guided Data-Construction agent that lifts a Dolly-only baseline by ~20 points on Llama-3.1-8B Finance, plus a Distributional Alignment Score evaluator that leads cross-model correlation in four of six domains. **Why it matters:** Turns "data quality" into a measurable, downstream-grounded target rather than surface heuristics.

- **OpenForgeRL: Train Harness-native Agents in Any Environment** — Yu et al. · https://arxiv.org/abs/2607.21557
  OpenForgeRL is an open-source framework for training harness-based agents (Claude Code / Codex-style) end-to-end, using a lightweight proxy that serves the harness's model calls while recording them as RL training data, plus a Kubernetes orchestrator running each rollout in its own container. With only hundreds to a few thousand tasks it reaches strong scores on ClawEval, OSWorld-Verified, Online-Mind2Web, and WebVoyager, matching or beating larger models in the GUI setting. Analysis shows harness choice strongly shapes learnability and that error recovery remains a weak spot. **Why it matters:** Lets researchers train agents directly inside the real harnesses they deploy in, closing a major train/inference gap.

- **TeaRAG: A Token-Efficient Agentic Retrieval-Augmented Generation Framework** — Zhang et al. · https://arxiv.org/abs/2511.05385
  TeaRAG compresses both retrieval content and reasoning steps in agentic RAG. It augments chunk-based retrieval with concise triplet graph retrieval and Personalized PageRank to surface key knowledge with fewer tokens, and uses an iterative process-aware DPO whose reward penalizes excessive reasoning steps. Across six datasets it improves Exact Match by 2–4% while cutting output tokens by ~60% on Llama3-8B and Qwen2.5-14B. **Why it matters:** Directly attacks the token-overhead tax that makes multi-round agentic RAG expensive in production.

- **AREX: Towards a Recursively Self-Improving Agent for Deep Research** — Lu et al. · https://huggingface.co/papers/2607.21461
  AREX is a family of recursively self-improving deep-research agents that alternate an inner research loop with an outer loop that audits the answer constraint-by-constraint and launches targeted follow-up research. It learns an autonomous context-update tool that compresses growing history into a compact state preserving verified evidence and unresolved constraints, trained via agentic mid-training and long-horizon RL. Across BrowseComp, WideSearch, HLE and others, its 4B and 122B-A10B MoE models beat comparable-scale baselines. **Why it matters:** Exploits the discovery-vs-verification asymmetry to make long-horizon research agents improve their own answers instead of just searching longer.

- **Human-in-the-Loop LLM Framework for Identification of Cutaneous Immune-Related Adverse Events** — Lu et al. · https://arxiv.org/abs/2607.20428
  A retrieval-augmented, multi-agent, human-in-the-loop LLM framework for detecting cutaneous immune-related adverse events from clinical notes. Versus unassisted manual review it improved F1 (0.88 vs 0.77) and inter-rater agreement (kappa 0.82 vs 0.50) while roughly halving review time. **Why it matters:** A concrete template for accurate, scalable, auditable adverse-event extraction that keeps clinicians in the loop.

- **LLM-INSTRUCT at UZH Shared Task 2026: Constraint-Aware Retrieval and Selective Debate for Paragraph-Level Argument Mining** — Tran et al. · https://arxiv.org/abs/2607.20430
  The winning system for the UZH ArgMining 2026 shared task frames paragraph-level argument mining as constrained structured prediction using only open-weight models up to 8B. It narrows the 141-tag space via metadata-aware dense retrieval, applies constrained decoding with per-dimension caps, escalates only uncertain cases to a three-agent debate branch, then validates the JSON schema — ranking 1st overall. **Why it matters:** Reinforces that shrinking the decision space before generation improves both accuracy and output robustness for small open models.

- **REFACT: Adaptive Fact Restatement for Compact and Faithful Chain-of-Thought Reasoning** — Jin et al. · https://arxiv.org/abs/2607.20833
  REFACT trains models to decide when a reasoning step needs contextual grounding and at what granularity to restate source facts, turning citations into answer-supporting intermediate states rather than post-hoc attachments. Optimized with a two-stage SFT-to-RL pipeline and a citation-utility reward, it improves long-context QA and counterfactual faithfulness on LongBench, LV-Eval, and ConFiQA while cutting token consumption. **Why it matters:** Produces reasoning traces that are denser rather than longer, improving faithfulness and cost at once.

## 📄 Research Papers — Top Picks
- **LakeQuest: A Three-Domain Benchmark for Grounded Question Answering across Data Lakes** — Solodko, Gong, Yu, Gorti, Cresswell, Zhong · https://arxiv.org/abs/2607.12310
  Most QA benchmarks hand the system a clean, schema-aligned corpus; real enterprise and scientific knowledge lives in messy data lakes of heterogeneous tables, passages, and linked metadata. LakeQuest is a human-validated set of 9,846 QA pairs spanning three domains, built to test the full retrieve-and-synthesize pipeline including the noisy discovery step. It refuses to abstract away the part where the system must find the right artifacts before reasoning over them. **Why it matters:** Agentic RAG systems fail most often at discovery, and this benchmark finally scores that stage end-to-end.

- **Keep Policy Gradient in Charge: Sibling-Guided Credit Distillation for Long-Horizon Tool-Use Agents** — Ding, Xin, De la Cruz Weinstein · https://arxiv.org/abs/2606.12634
  Long-horizon tool-use RL learns from outcome verification, so a single trajectory-level advantage gets smeared across reasoning, API, and answer tokens alike. Naive self-distillation densifies the signal but can destroy tool use by rehearsing teacher behavior without identifying which actions the verifier actually rewarded. SGCD instead uses distillation only for bounded credit weighting: mixed successful/failed sibling rollouts are contrasted by an external LLM into a training-only credit signal while policy gradient stays the actor. **Why it matters:** A targeted fix for the credit-assignment problem that makes multi-step tool agents so expensive to train.

- **Transforming LLMs into Efficient Cross-Encoders via Knowledge Distillation for RAG Reranking** — Dasa Lakshminath, Shubhan S · https://arxiv.org/abs/2607.11933
  Cross-encoders rerank accurately but their quadratic inference cost blocks real-time use. The authors fine-tune LLaMA 3 8B as a drop-in reranker via supervised fine-tuning on a custom relevance dataset with LoRA adapters, then 4-bit quantize it for serving. The result slots into a dual-retriever pipeline combining BM25 and dense vector search, evaluated with RAGAS on a domain-specific QA benchmark. **Why it matters:** A practical recipe for cross-encoder-grade reranking at latency budgets production RAG can actually afford.

- **SkillSelect-Serve: QoS-Aware Budgeted Skill Service Recommendation for LLM Agents** — Zheng, Wang, Zhang, Chen, Li, et al. · https://arxiv.org/abs/2607.00011
  Reusable agent skills are becoming a service layer, but a skill is not a plain retrieval item: it carries capabilities, I/O assumptions, tool dependencies, context cost, and risk metadata. Small agents can only load a few capability units under tight context, tool availability, and risk limits, yet fixed Top-k methods rank skills by textual relevance alone. SkillSelect-Serve performs QoS-aware, budget-constrained skill selection that accounts for requirement satisfaction and deliverability. **Why it matters:** Directly addresses skill/tool selection under context budget — the bottleneck for small-model agents.

- **PalmClaw: A Native On-Device Agent Framework for Mobile Phones** — Cai, Li, Wei, Li · https://arxiv.org/abs/2607.13027
  Mobile agents today mostly drive phones through GUI actions — tapping, swiping, typing — producing long, brittle, interface-dependent sequences that cannot reach device capabilities directly. PalmClaw proposes a native on-device agent framework that calls device capabilities rather than pantomiming a human finger. The phone is argued to be a first-class agent environment because it holds the user's data, sensors, and daily apps. **Why it matters:** Native capability calls could make mobile agents dramatically shorter-horizon and more reliable than GUI-scripted ones.

- **Function-Aware Fill-in-the-Middle as Mid-Training for Coding Agent Foundation Models** — Wang, Liang, Zhang, Liu, Wei, Zhang, Nie, Chen · https://arxiv.org/abs/2607.12463
  A coding agent's action-observation-continuation loop is structurally isomorphic to a function call site: a caller binds arguments, a callee returns a value computed elsewhere, downstream code consumes it. Standard left-to-right pretraining only ever exposes the forward direction, so the authors mid-train with function-aware fill-in-the-middle, masking functions selected via program dependency graphs. The conditioning structure they exploit already exists at internet scale in ordinary code. **Why it matters:** Turns free public code into supervision for the exact skill agents need — folding a tool return into ongoing reasoning.

- **Who Grades the Grader? Co-Evolving Evaluation Metrics and Skills for Self-Improving LLM Agents** — Zhang, Wang, Cui, Li, Qiu, Zhu, He · https://arxiv.org/abs/2607.12790
  Self-evolving agents create, revise, and retire their own skills — but every such loop silently assumes a reliable evaluation metric already exists, which in most real applications it does not. The paper evolves the metric itself: a loop searches compositions of small drawback detectors under a full evolutionary lifecycle, anchored to a ten-item reference set, regularized by consensus over unlabeled outputs, and audited against a held-out anchor it never reads. The output is a transparent, inspectable metric rather than an opaque judge. **Why it matters:** Without a trustworthy grader, self-improvement loops optimize noise — this makes the grader a first-class evolving artifact.

- **MemOps: Benchmarking Lifecycle Memory Operations in Long-Horizon Conversations** — Hao, Zhang, Lin, Sun, Guo, Zhang, Liang, Xiong, Li · https://arxiv.org/abs/2607.12893
  Existing memory benchmarks judge agents only by final-answer correctness on downstream QA, conflating very different failure causes: never storing a fact, binding an operation to the wrong target, or serving stale values after a correction. That black-box framing can even credit correct answers that rested on inconsistent or unsafe memory state. MemOps instead benchmarks the memory lifecycle operations themselves across extended multi-session interactions. **Why it matters:** Makes agent memory debuggable by attributing failures to the specific operation that broke.

- **Do AI Agents Know When a Task Is Simple? Toward Complexity-Aware Reasoning and Execution** — Yin, Feng · https://arxiv.org/abs/2607.13034
  LLM agents rarely ask how much effort a task needs, defaulting to a maximum-context-first strategy that re-reads files and dependencies already seen — turning a one-line edit into a small codebase audit. The authors formalize minimum-sufficient execution plus an Agent Cognitive Redundancy Ratio (ACRR), and propose E3 (Estimate, Execute, …) to judge difficulty and the shortest reliable path before committing budget. **Why it matters:** Scope estimation is where most agent token spend quietly evaporates.

- **A Learning-Rate-Gated Failure of GRPO in a Small Language and Vision-Language Model Web Agent** — Gan, Cai, Liang, Wei, Ni, Zhang · https://arxiv.org/abs/2607.12640
  GRPO is now routinely run on a supervised checkpoint hoping to produce a stronger agent; this paper asks whether it adds skill at the 4B–8B scale or merely reshapes behavior the supervised model already had. Across a controlled grid of 18 runs varying learning rate, KL weight, seed, initialization, and clipping, no configuration credibly improved success rate over a strong supervised baseline on tasks the agent had largely mastered. It is published explicitly as a controlled null with a proposed mechanism. **Why it matters:** A rare negative result that should temper default "just add RL" assumptions for small web agents.

- **Declarative by Design, Assistable Only by Convention: Benchmarking Multi-Agent Frameworks for AI-Assistability** — Ahmed, Deshpande · https://arxiv.org/abs/2602.11198
  Multi-agent frameworks promise simpler LLM-driven development, but nothing measures how well coding assistants can actually write correct framework-specific code. The authors introduce AI-assistability, combining structural alignment with functional correctness (pass@1), and evaluate it via DDL2PropBank — mapping relational DB schemas to PropBank rolesets with identical agent logic implemented across ten frameworks. Declarative design turns out not to guarantee assistability. **Why it matters:** Framework choice measurably changes how well your coding agent can use it — now with a metric.

- **Catalyst-Agent: Autonomous heterogeneous catalyst screening with an LLM Agent** — Chandrasekhar, Ock, Barati Farimani · https://arxiv.org/abs/2603.01311
  MLIPs and GNNs already made individual adsorption-energy calculations orders of magnitude cheaper than DFT, but large-scale screening still stalls on human decisions: picking candidates, constructing slabs, enumerating adsorption sites, interpreting descriptor failures, choosing follow-up modifications. Catalyst-Agent puts an LLM agent in that decision loop for ORR, NRR, and CO2RR screening. **Why it matters:** A clean case of agents removing the human-in-the-loop bottleneck in a real scientific pipeline rather than a toy benchmark.

- **Tracing Agentic Failure from the Flow of Success** — Yeh, Zhu, Deep, Li · https://arxiv.org/abs/2607.12747
  Failure attribution — pinpointing which step in a trajectory doomed the task — currently needs either expensive prompting pipelines or post-training on failure traces with step-level error labels that are costly to collect and hard to scale. This work does unsupervised failure attribution, training exclusively on successful trajectories with no step-level supervision on failure data. The result is a lightweight, trainable attribution model. **Why it matters:** Makes agent debugging tractable without hand-annotating every way a run can go wrong.

- **KnowAct-GUIClaw: Personal GUI Assistant with Self-Evolving Memory and Skill** — Li, Li, Su, Xu, Zhao, Bian, Liang, Zhang, Hu, Zhang · https://arxiv.org/abs/2607.12625
  OpenClaw leads on complex task automation but lacks cross-platform GUI support and a real self-evolution mechanism, limiting adaptation across device ecosystems and preventing improvement from execution experience. The Know Deeply, Act Perfectly paradigm holds that accumulated user interaction and task-running experience directly improve accuracy and efficiency, unifying cognitive comprehension with operational execution. The system builds self-evolving memory and skill on that premise. **Why it matters:** Assistants that compound experience across sessions are the natural next step past stateless GUI scripting.

- **A JoLT for the KV Cache: Near-Lossless KV Cache Compression via Joint Tucker and JL-Residual Allocation** — Krishnan, Schulz · https://arxiv.org/abs/2607.12550
  The KV cache, not the weights, sets the throughput ceiling at long context, growing with batch size, context length, and depth. Existing low-rank methods factor 2D slices and quantization lowers bit-width uniformly — neither exploits that a layer's cache is a third-order tensor whose head, token, and feature axes carry very different redundancy. JoLT applies joint Tucker decomposition plus a JL-residual allocation across those axes. **Why it matters:** Long-context agents live or die on KV cache economics, and axis-aware compression is the underexploited lever.

## 📄 Research Papers — Top Picks

- **From Chatbot to Digital Colleague: The Paradigm Shift Toward Persistent Autonomous AI** — arXiv cs.AI · https://arxiv.org/abs/2606.14502
  Surveys the shift of LLMs from conversational chatbots to 'digital colleagues' - persistent, autonomous systems with reasoning, action, memory, and self-improvement. It organizes the transition along a cognitive-core axis and an autonomy axis, framing the move from one-shot answers to durable, ongoing work. A useful conceptual map of where agent systems are heading. **Why it matters:** see summary line above.
- **An Empirical Study of Automating Agent Evaluation** — arXiv cs.CL · https://arxiv.org/abs/2605.11378
  Studies whether frontier coding assistants can automate the costly job of evaluating agents on multi-step tool-use and reasoning tasks. Naively prompting them is insufficient: without domain-specific evaluation knowledge they hit only ~30% execution success. The paper diagnoses the gap and points toward knowledge-grounded auto-evaluation. **Why it matters:** see summary line above.
- **SkillAudit: Ground-Truth-Free Skill Evolution via Paired Trajectory Auditing** — arXiv cs.AI · https://arxiv.org/abs/2606.14239
  SkillAudit evolves agent skill packages without ground-truth feedback by auditing paired execution trajectories. Instead of relying on held-out scores or hidden test outcomes, it compares trajectories to surface where a skill fails after deployment. Addresses the real problem that skills degrade as APIs and edge cases shift. **Why it matters:** see summary line above.
- **PRISM: Perception Reasoning Interleaved for Sequential Decision Making** — arXiv cs.AI · https://arxiv.org/abs/2605.05407
  PRISM tightly couples a Vision-Language Model (perception) and an LLM (decision) through a dynamic question-answer pipeline for sequential decision-making. Rather than passively captioning a scene, the LLM actively queries the VLM for task-critical details, closing the perception-reasoning gap in embodied multimodal agents. **Why it matters:** see summary line above.
- **EvoTrainer: Co-Evolving LLM Policies and Training Harnesses for Autonomous Agentic Reinforcement Learning** — arXiv cs.AI · https://arxiv.org/abs/2606.03108
  EvoTrainer co-evolves the LLM policy AND the training harness for agentic RL, instead of fixing the harness and only searching recipes. It diagnoses rollout-level evidence, revises its own diagnostics, and backtests interventions to handle shifting bottlenecks that scalar rewards hide. **Why it matters:** see summary line above.
- **AgentCyberRange: Benchmarking Frontier AI Systems in Realistic Cyber Ranges** — arXiv cs.AI · https://arxiv.org/abs/2606.14295
  AgentCyberRange benchmarks frontier AI on realistic multi-host cyber ranges, going beyond isolated CTF or single-exploit tasks to capture full intrusion workflows. It provides reproducible environments to measure offensive capabilities like recon, vuln detection, and exploitation end-to-end. **Why it matters:** see summary line above.
- **CacheRL:Multi-Turn Tool-Calling Agents via Cached Rollouts and Hybrid Reward** — arXiv cs.CL · https://arxiv.org/abs/2606.14179
  CacheRL trains small agent models to 92% process accuracy on multi-step tool-calling - near GPT-5's 94% - with ~100x less compute. It transfers tool-use knowledge from large models, runs RL via cached rollouts (no costly live tool execution), and learns robustly from noisy signals. **Why it matters:** see summary line above.
- **SIMMER: Benchmarking Latent Failures in LLM Executable Planning with a World Model** — arXiv cs.CL · https://arxiv.org/abs/2606.14574
  SIMMER benchmarks 'latent failures' in LLM planning - plans that execute without errors yet silently fail to achieve the goal - using a world model to detect them. Unlike benchmarks that only check whether a plan runs, it catches the harder, quieter failure mode in household-agent settings. **Why it matters:** see summary line above.
- **GitOfThoughts: Version-Controlled Reasoning and Agent Memory You Can Replay, Diff, and Merge** — arXiv cs.CL · https://arxiv.org/abs/2606.14470
  GitOfThoughts stores an agent's reasoning tree as a git repository: every scored thought is a commit, so chains of thought become replayable, diffable, mergeable, and auditable. It brings version control to reasoning and agent memory, which today vanish with the context window. **Why it matters:** see summary line above.
- **Verbatim Chunks Beat Extracted Artifacts: A Controlled Ablation of Memory Representations for Long LLM Conversations** — arXiv cs.CL · https://arxiv.org/abs/2601.00821
  A controlled ablation of conversational-memory representations: it swaps only the stored form - LLM-extracted typed artifacts (facts/decisions) versus verbatim conversation chunks - inside one fixed retrieval-rerank pipeline. Verbatim chunks beat distilled artifacts, challenging the common assumption that structured memory retrieves better. **Why it matters:** see summary line above.
- **Hidden in Plain Sight: Benchmarking Agent Safety Against Decomposition Attacks with DECOMPBENCH** — arXiv cs.AI · https://arxiv.org/abs/2606.13994
  DECOMPBENCH benchmarks agent safety against decomposition attacks, where a harmful goal is split into benign-looking subtasks that each pass safety checks but together fulfill malicious intent. It exposes a blind spot in per-step safety filtering for multi-step agents. **Why it matters:** see summary line above.
- **Doctor-RAG: A Failure-Aware Repair Framework for Agentic Retrieval-Augmented Generation** — arXiv cs.IR · https://arxiv.org/abs/2604.00865
  Doctor-RAG (DR-RAG) is a diagnose-and-repair framework for agentic retrieval-augmented generation. Instead of stopping at diagnosis or doing coarse replanning, it localizes the specific error in a long retrieval-reasoning trajectory and repairs it, cutting the compute of rerun-style recovery. **Why it matters:** see summary line above.
- **TVIR: Building Deep Research Agents Towards Text-Visual Interleaved Report Generation** — arXiv cs.CL · https://arxiv.org/abs/2606.02320
  TVIR builds deep-research agents that generate text-visual interleaved reports and introduces TVIR-Bench to test whether embedded visuals are factually reliable and aligned with the surrounding analysis. It pushes research agents past text-only output toward verifiable multimodal reporting. **Why it matters:** see summary line above.
- **Optimizing Agentic Reasoning with Retrieval via Synthetic Semantic Information Gain Reward** — arXiv cs.AI · https://arxiv.org/abs/2602.00845
  InfoReasoner optimizes agentic retrieval with a synthetic 'semantic information gain' reward, redefining information gain as uncertainty reduction. It gives dense, principled reward signals for when an agent should seek external knowledge, addressing the sparse-reward problem in retrieval agents. **Why it matters:** see summary line above.
- **Orchestra-o1: Omnimodal Agent Orchestration** — arXiv cs.CL · https://arxiv.org/abs/2606.13707
  Orchestra-o1 tackles omnimodal agent orchestration, coordinating heterogeneous modalities that coexist and interact, beyond the narrow modality sets existing frameworks support. It targets task decomposition and multi-agent collaboration in genuinely multimodal settings. **Why it matters:** see summary line above.

## 📄 Research Papers — Top Picks
- **Declarative Skills for AI Agents in Knowledge-Grounded Tool-Use Workflows** — Lim et al. · https://arxiv.org/abs/2606.06923
  Studies orchestration for tool-using agents over unstructured knowledge bases, arguing that "declarative" agents equipped with natural-language skill files appended to the system prompt are an effective orchestration paradigm for customer-service workflows. **Why it matters:** Directly validates the skill-file pattern (à la Claude Skills) as a structured way to govern tool-use agents.
- **Towards On-Policy Data Evolution for Visual-Native Multimodal Deep Search Agents** — Huang et al. · https://arxiv.org/abs/2605.10832
  Tackles two bottlenecks in multimodal deep-search agents that chain search, tool use, and visual reasoning over evolving text+image context, proposing on-policy data evolution to improve tool-use harnesses. **Why it matters:** Pushes agents toward treating images as first-class search/tool outputs rather than afterthoughts.
- **Queen-Bee Agents: A BeeSpec-Centered Architecture for Governed Enterprise MCP Orchestration** — Zhang & Liaotian · https://arxiv.org/abs/2606.06545
  Proposes an architecture connecting LLMs to private tools and Model Context Protocol interfaces with policy enforcement, tenant-scoped isolation, and governance for enterprise agent systems. **Why it matters:** Addresses the governance/isolation gap that blocks MCP adoption in real enterprises.
- **AutoTool: Dynamic Tool Selection and Integration for Agentic Reasoning** — Zou et al. · https://arxiv.org/abs/2512.13278
  Moves beyond fixed tool inventories by letting agentic-RL LLMs dynamically select and integrate new/evolving tools during long chain-of-thought tool use. **Why it matters:** Adaptive tool discovery is key to agents that survive changing toolsets in production.
- **Exploring Agentic Tool-Calling Decisions via Uncertainty-Aligned Reinforcement Learning** — Zhou et al. · https://arxiv.org/abs/2606.06976
  Reduces unsupported tool invocations and hallucinated direct answers by aligning tool-calling decisions with model uncertainty via RL. **Why it matters:** Targets a core reliability failure mode in multi-step agent execution.
- **From Privacy to Workflow Integrity: Communication-Graph Metadata in Autonomous Agent Interoperability** — Dangol · https://arxiv.org/abs/2606.07150
  Observes that A2A and MCP protect message content but leave the agent communication graph (who talks to whom) exposed over HTTP(S), and examines the resulting integrity/privacy risks. **Why it matters:** Flags an under-appreciated metadata attack surface in standardized agent interop.
- **Beyond tokens: a unified framework for latent communication in LLM-based multi-agent systems** — Liu · https://arxiv.org/abs/2606.05711
  Proposes latent (non-token) communication between LLM agents instead of verbalized natural-language messages, unifying approaches under one framework. **Why it matters:** Latent inter-agent channels could cut multi-agent coordination cost and information loss.
- **MADRAG: Multi-Agent Debate with Retrieval-Augmented Generation for Training-Free Analytic Essay Scoring** — Keramati et al. · https://arxiv.org/abs/2606.06754
  Combines multi-agent debate with RAG grounding for training-free essay scoring, decomposing LLM-as-judge into an interactive, less biased process. **Why it matters:** Shows multi-agent debate + retrieval as a practical recipe to stabilize LLM evaluation.
- **SpatialWorld: Benchmarking Interactive Spatial Reasoning of Multimodal Agents in Real-World Tasks** — Gao et al. · https://huggingface.co/papers/2606.09669
  Introduces an interactive benchmark for spatial reasoning of multimodal agents, moving past passive VQA and simulator-specific pipelines. **Why it matters:** Better interactive benchmarks are needed to measure agents that actually act in the physical world.
- **MemDreamer: Decoupling Perception and Reasoning for Long Video Understanding** — Chen et al. · https://arxiv.org/abs/2606.07512
  Decouples perception from reasoning via hierarchical graph memory and agentic retrieval to handle hours-long video without token explosion. **Why it matters:** Memory + agentic retrieval is a promising route to long-context multimodal understanding.
- **M³Exam: Benchmarking Multimodal Memory for Realistic User-Agent Interactions** — Huang et al. · https://arxiv.org/abs/2606.07402
  Benchmarks agents reasoning over accumulating multimodal information in realistic user-agent settings, beyond sparse-visual human-human assumptions. **Why it matters:** Tests memory capabilities agents actually need in long-running deployments.
- **SlideAgent: Hierarchical Agentic Framework for Multi-Page Visual Document Understanding** — Jin et al. · https://arxiv.org/abs/2510.26615
  A hierarchical agentic framework for understanding multi-page visual documents (manuals, decks, posters) that exploit layout and cross-page references. **Why it matters:** Document agents that grasp layout unlock enterprise knowledge trapped in slides/PDFs.
- **MCERF: Advancing Multimodal LLM Evaluation of Engineering Documentation with Enhanced Retrieval** — Naghavi Khanghah et al. · https://arxiv.org/abs/2604.09552
  Improves RAG over engineering rulebooks containing dense text, tables, and illustrations, extending the DesignQA framework with enhanced multimodal retrieval. **Why it matters:** Multimodal RAG on technical standards is high-value for regulated engineering domains.
- **LLM Agent-Assisted Reverse Engineering with Quantitative Readability Metrics** — Archibald & Thijssen · https://arxiv.org/abs/2606.06838
  Uses LLM agents guided by quantitative readability metrics to make functionally-correct-but-unreadable decompiled C code readable. **Why it matters:** Agentic, metric-guided code transformation is a concrete win for security/RE workflows.
- **Agentic Physical AI toward a Domain-Specific Foundation Model for Energy Systems** — Lee et al. · https://arxiv.org/abs/2512.23292
  Argues general-purpose foundation models hit a control-interface barrier (50–53% on basic quantitative physics) and proposes a domain-specific agentic approach, with a nuclear-reactor-control case study. **Why it matters:** Highlights limits of general models at physical control and the case for domain-specialized agents.
